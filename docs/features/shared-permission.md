# 共享权限功能规格

本文档定义 Coremail 日历模块中共享权限的产品语义、入口、交互、文案和验收规则。后续实现和评审优先引用本文。

上游依据：

- `docs/product.md`
- `docs/design.md`
- `docs/information-architecture.md`
- `docs/components.md`

## 1. 功能定位

共享权限用于解决企业日历协作中的两个问题：

- 共享方：把自己拥有的日历共享给别人，并控制对方权限。
- 接收方：查看别人共享给我的日历权限，并决定是否保留在自己的列表中。

该功能必须明确区分：

- 删除我拥有的日历。
- 从我的列表移除别人共享给我的日历。
- 取消我共享给别人的权限。
- 修改别人对我日历的访问权限。

共享权限不属于设置模块的低频配置；它是日历模块左侧日历列表中的日常管理能力。

## 2. 角色与对象

### 2.1 共享方

共享方是日历拥有者。

共享方可以：

- 将我的日历共享给成员或团队。
- 设置权限。
- 查看共享对象状态。
- 修改权限。
- 取消共享。

共享方不能通过“从列表移除”管理接收方访问；该语义只属于接收方本地列表。

### 2.2 接收方

接收方是收到别人共享日历的人。

接收方可以：

- 接收共享邀请。
- 在左侧看到共享日历。
- 查看自己拥有的权限。
- 控制共享日历是否显示在主视图。
- 从自己的列表移除共享日历。

接收方不能：

- 删除共享方的日历。
- 管理共享方的共享记录。
- 将该共享日历再次共享给别人。
- 修改自己被授予的权限。

### 2.3 我的日历

我的日历是当前用户拥有并可管理的日历。

菜单：

1. 颜色
2. 重命名
3. 共享与权限
4. 导出
5. 删除日历

### 2.4 共享日历

共享日历是别人共享给当前用户的日历。

菜单：

1. 颜色
2. 查看权限
3. 从列表移除

共享日历不展示：

- 设置
- 共享与权限
- 重命名
- 导出
- 删除日历

除非未来业务明确支持对应能力。

## 3. 权限模型

### 3.1 权限类型

| 权限 | 可见能力 | 不可做 |
| --- | --- | --- |
| 仅查看详情 | 查看日程标题、时间、地点、参与人、正文或备注 | 新建、编辑、删除日程；修改提醒、分类、忙碌状态 |
| 可编辑 | 查看详情；新建、编辑、删除日程；修改提醒、分类、忙碌状态 | 修改共享权限；再次分享；管理共享成员 |
| 可查看闲忙 | 仅查看忙闲占用、时间块、时间分布 | 查看标题、地点、正文、附件、参与人；编辑或删除日程 |

### 3.2 权限标签

权限标签统一为：

- 仅查看详情
- 可编辑
- 可查看闲忙

不要混用：

- 仅查看
- 查看所有详细信息
- 查看标题和地点
- 闲忙

如果底层数据存在旧值，界面层应映射到统一标签。

### 3.3 权限说明

查看权限或选择权限时，必须提供简短说明。

推荐文案：

- 仅查看详情：可查看该日历中的详细事件信息，不可编辑。
- 可编辑：可查看并编辑该日历中的事件，不可管理共享权限。
- 可查看闲忙：仅可查看忙闲时间，不可查看详细事件内容。

## 4. 分享对象状态

### 4.1 状态定义

| 状态 | 含义 |
| --- | --- |
| 待接收 | 共享已发起，对方尚未确认 |
| 已接收 | 对方已确认接收，正常使用中 |
| 已失效 | 共享被取消、权限失效或对象不可用 |

### 4.2 状态展示

状态只在共享方管理共享记录时展示。

接收方查看自己共享日历时，重点展示当前权限，不强调共享记录状态。

### 4.3 状态变更

状态变更来源：

- 共享方发起共享：生成待接收记录。
- 接收方接收：状态变为已接收。
- 共享方取消共享：状态变为已失效或记录移除。
- 接收方从列表移除：不改变共享方记录状态。

## 5. 入口规则

### 5.1 左侧日历项

共享权限入口都从左侧日历项进入。

日历项包含：

- 显隐勾选区。
- 颜色标识。
- 日历名称。
- 更多菜单入口。
- 桌面端右键菜单。

### 5.2 单击行为

| 操作位置 | 行为 |
| --- | --- |
| 勾选区 | 控制该日历是否显示在主画布 |
| 日历项主体 | 聚焦或选中该日历项，不打开管理面板 |
| 更多菜单 | 打开该日历可用操作 |
| 右键 | 打开与更多菜单一致的操作菜单 |

### 5.3 Hover 菜单与右键菜单

同一对象的 hover 更多菜单和右键菜单内容必须一致。

右键菜单是桌面端效率增强，不是唯一入口。

## 6. 我的日历：共享与权限

### 6.1 入口

路径：

```text
左侧我的日历 -> 更多菜单或右键菜单 -> 共享与权限
```

### 6.2 面板内容

共享与权限面板包含：

- 日历名称。
- 标题：共享与权限。
- 添加共享对象入口。
- 权限选择。
- 已共享对象列表。
- 每个对象的权限、状态和操作。

### 6.3 添加共享对象

规则：

- 输入成员、团队或邮箱。
- 必须选择权限后才能确认。
- 确认后生成共享记录。
- 默认状态为待接收。
- 可通过通知或邮件让对方接收。

### 6.4 已共享列表

每行展示：

- 成员名称或团队名称。
- 邮箱或部门等辅助识别信息。
- 当前权限。
- 接收状态。
- 修改权限。
- 取消共享。

人员识别优先展示姓名、部门或职位，不以头像作为主要识别信息。

### 6.5 修改权限

共享方可修改已共享对象权限。

规则：

- 修改权限后立即更新。
- 已接收对象立即生效。
- 待接收对象在对方接收后按最新权限生效。
- 成功后给出轻提示。

推荐提示：

```text
日历共享权限已更新
```

或：

```text
已更新待接收权限，对方接受后生效
```

### 6.6 取消共享

共享方可取消某个对象的共享权限。

路径：

```text
共享与权限 -> 已共享列表 -> 取消共享
```

规则：

- 必须二次确认。
- 文案表达“撤销访问权限”，不要写“从列表移除”。
- 确认后该对象失去访问权限。
- 接收方的共享日历应失效、消失或置灰。

确认框应说明：

- 将影响对方访问。
- 对方将无法继续查看或编辑该日历。
- 不会删除我的日历本身。

## 7. 接收共享

### 7.1 来源

接收方可能通过以下入口收到共享：

- 站内通知。
- 邮件通知。
- 消息中心。
- 添加共享日历流程。

### 7.2 通知内容

通知应展示：

- 共享方。
- 日历名称。
- 权限类型。
- 接收或忽略操作。

### 7.3 接收后行为

接收共享后：

- 加入左侧共享日历区域。
- 默认可见。
- 保留共享方或来源账号信息。
- 可通过菜单查看权限。
- 给出成功提示。

推荐提示：

```text
已添加到共享日历
```

## 8. 共享日历：查看权限

### 8.1 入口

路径：

```text
左侧共享日历 -> 更多菜单或右键菜单 -> 查看权限
```

### 8.2 面板内容

查看权限面板包含：

- 日历名称。
- 共享方信息。
- 当前权限标签。
- 权限说明。
- 权限能力列表。

### 8.3 能力说明

能力说明按当前权限展示。

仅查看详情：

- 可查看标题。
- 可查看时间。
- 可查看地点。
- 可查看正文或备注。
- 不可编辑。

可编辑：

- 可查看详情。
- 可新建、编辑、删除日程。
- 不可管理共享权限。
- 不可再次分享。

可查看闲忙：

- 只显示忙闲占用。
- 不显示标题、地点、正文和参与人。
- 不可编辑。

### 8.4 禁止操作

接收方查看权限面板不提供：

- 修改权限。
- 申请权限。
- 再次分享。
- 取消共享。

如果未来要做申请权限，应作为独立功能评估。

## 9. 共享日历：从列表移除

### 9.1 入口

路径：

```text
左侧共享日历 -> 更多菜单或右键菜单 -> 从列表移除
```

### 9.2 语义

从列表移除表示：

- 只从我的共享日历列表中移除。
- 只影响我的当前视图和列表。
- 不删除共享方的日历。
- 不影响共享方的权限设置。
- 不等同于取消共享。

### 9.3 确认框

标题：

```text
从列表移除该共享日历？
```

说明：

```text
这只会从你的共享日历列表中移除，不会影响共享方的日历或权限设置。
```

如果能展示具体对象：

```text
这只会把 {日历名称} 从你的共享日历列表和当前视图中移除，不会影响共享方的日历或权限设置。
```

按钮：

- 取消
- 从列表移除

### 9.4 移除后行为

确认后：

- 从左侧共享日历列表移除。
- 从主视图可见日历中移除。
- 如果该日历正在被聚焦，清除聚焦状态。
- 如果权限面板正在展示该日历，关闭权限面板。
- 给出轻提示。

推荐提示：

```text
已移除 {日历名称}
```

### 9.5 不实现的新模型

除非产品明确要求，不实现复杂的“本地隐藏但可重新加入”状态模型。

如果未来需要重新加入能力，应定义：

- 从哪里重新加入。
- 是否需要共享方重新授权。
- 接收方是否能看到历史移除记录。
- 共享方是否能看到接收方移除状态。

## 10. 菜单与文案

### 10.1 我的日历菜单

顺序：

1. 颜色
2. 重命名
3. 共享与权限
4. 导出
5. 删除日历

危险项：

- 删除日历放在末尾。
- 使用危险色。
- 必须二次确认。

### 10.2 共享日历菜单

顺序：

1. 颜色
2. 查看权限
3. 从列表移除

规则：

- hover 菜单和右键菜单一致。
- 不展示设置入口。
- 不展示删除入口。
- 不展示共享与权限入口。

### 10.3 文案对照

| 场景 | 推荐文案 | 禁止或慎用文案 |
| --- | --- | --- |
| 我的日历共享管理 | 共享与权限 | 设置、管理、分享 |
| 接收方查看权限 | 查看权限 | 设置、共享权限 |
| 接收方移除共享日历 | 从列表移除 | 删除、删除共享日历、取消共享 |
| 共享方撤销访问 | 取消共享 | 从列表移除、删除权限 |
| 删除我拥有的日历 | 删除日历 | 从列表移除 |

## 11. 权限对日历内容的影响

### 11.1 仅查看详情

可见：

- 标题。
- 时间。
- 地点。
- 参与人。
- 正文或备注。

不可做：

- 新建日程。
- 编辑日程。
- 删除日程。
- 修改提醒、分类、忙碌状态。

### 11.2 可编辑

可做：

- 查看详情。
- 新建日程。
- 编辑日程。
- 删除日程。
- 修改提醒、分类、忙碌状态。

不可做：

- 修改共享权限。
- 再次分享。
- 管理共享成员。

### 11.3 可查看闲忙

可见：

- 忙闲占用。
- 时间块。
- 时间分布。

不可见：

- 标题。
- 地点。
- 正文。
- 附件。
- 参与人。

不可做：

- 查看详情。
- 编辑。
- 删除。
- 入会。

## 12. 边界与反模式

### 12.1 不支持申请更高权限

当前不支持接收方申请更高权限。

原因：

- 简化模型。
- 避免引入审批流。
- 避免让共享权限面板变成消息系统。

### 12.2 不支持共享日历本地重命名

共享日历默认不支持本地重命名。

原因：

- 避免接收方名称与共享方名称不一致。
- 降低来源识别成本。

### 12.3 不支持共享日历导出

共享日历默认不支持导出。

原因：

- 权限风险高。
- 容易导出超出接收方授权范围的数据。

### 12.4 反模式

避免：

- 把共享日历菜单写成 `设置`。
- 把接收方移除写成 `删除共享日历`。
- 把共享方取消共享写成 `从列表移除`。
- 让接收方管理共享权限。
- 让 hover 菜单和右键菜单不一致。
- 用头像作为共享对象的唯一识别。
- 在确认框中不说明影响范围。

## 13. 状态反馈

### 13.1 成功反馈

推荐：

- 已添加到共享日历
- 日历共享权限已更新
- 已更新待接收权限，对方接受后生效
- 已移除 {日历名称}
- 已取消对 {成员} 的共享

### 13.2 失败反馈

失败反馈要说明原因。

推荐：

- 当前共享日历仅可查看闲忙，无法执行此操作
- 该共享记录已失效
- 无法移除当前日历，请稍后重试

不推荐：

- 操作失败
- 出错了
- 请重试

## 14. 验收清单

实现或评审共享权限时，检查：

1. 我的日历和共享日历菜单是否区分清楚？
2. 共享日历菜单是否只有颜色、查看权限、从列表移除？
3. hover 更多菜单和右键菜单是否一致？
4. 共享日历是否没有设置、删除、共享与权限入口？
5. 从列表移除是否没有写成删除？
6. 从列表移除确认框是否说明不影响共享方日历或权限设置？
7. 共享方取消共享是否说明会影响对方访问？
8. 权限标签是否统一为仅查看详情、可编辑、可查看闲忙？
9. 可编辑权限是否明确不能管理共享权限？
10. 仅忙闲权限是否隐藏标题、地点、正文、附件和参与人？
11. 接收方查看权限面板是否不提供修改权限？
12. 共享对象列表是否展示对象名称、辅助识别信息、权限和状态？
13. 人员识别是否不依赖头像？
14. 危险操作是否有二次确认？
15. 操作成功或失败后是否有明确反馈？
16. 文案是否和 `docs/design.md`、`docs/components.md` 保持一致？
