# 邮件搜索结果页功能规格

本文档定义 Coremail 邮件模块中普通邮件搜索结果页的产品规则、交互行为和信息展示。它用于指导后续实现、评审和验收。

上游依据：

- `docs/product.md`
- `docs/design.md`
- `docs/information-architecture.md`
- `docs/components.md`

## 1. 页面定位

邮件搜索结果页是邮件模块内的普通检索状态。

用户进入该页通常是为了：

- 找到某封邮件或某组邮件。
- 判断邮件是否来自目标人员或目标账号。
- 查看主题、摘要、附件和时间。
- 打开邮件详情。
- 执行回复、转发、归档、标记等邮件操作。
- 从邮件中进入关联日程或生成日程。

该页面不是：

- 智能邮件助手。
- 自动摘要中心。
- 邮件统计看板。
- 高级搜索配置页。
- 任务管理系统。

普通邮件搜索不展示复杂字段配置，不解释系统搜索了哪些字段。

## 2. 用户场景

### 2.1 找到某封邮件

用户记得关键词、主题片段、发件人或附件名称。

页面应帮助用户快速确认：

- 这是不是目标邮件。
- 谁发来的。
- 发给了谁。
- 属于哪个账号和文件夹。
- 是否有附件。
- 最近一次发生在什么时候。

### 2.2 找到一组相关邮件

用户搜索项目名、客户名、合同号、预算、评审等关键词，希望找到相关邮件集合。

页面应：

- 使用高密度列表。
- 支持按时间、账号、文件夹、人员和附件筛选。
- 避免一次铺开过多结果。
- 支持通过会话或主题关联判断上下文。

### 2.3 找附件材料

用户通过附件名称或材料关键词搜索。

页面应：

- 在附件名称与关键词相关时展示附件信息。
- 不用大文件图标占空间。
- 附件打开、下载、预览等操作进入邮件详情处理。

### 2.4 从邮件转到日程

有些邮件包含会议邀请、排期需求或已关联日程。

页面应：

- 展示关联日程线索。
- 允许打开关联日程。
- 对排期意图只做轻提示，不把搜索结果变成智能助手。

## 3. 页面结构

推荐结构：

1. 邮件模块顶部工具栏中的搜索框。
2. 筛选栏。
3. 结果数量与筛选摘要。
4. 邮件结果列表。
5. 邮件详情阅读区。
6. 加载更多或显示全部。
7. 空状态。

邮件搜索可以保留邮件模块的列表 + 详情分栏，因为邮件任务天然包含“搜索列表中选择一封并阅读”的连续流程。

不建议：

- 做成卡片瀑布流。
- 做成日历式时间轴。
- 使用大面积统计卡片。
- 默认展开所有正文。
- 默认展示所有附件操作。

## 4. 搜索框

### 4.1 位置

搜索框位于邮件模块顶部工具栏。

规则：

- 搜索前后位置保持一致。
- 搜索结果页不另起一个主体大搜索框。
- 搜索状态可显示 `返回邮件` 或 `返回收件箱`。
- 搜索框尺寸不因进入结果页突然变长。

### 4.2 展示

搜索框展示当前关键词。

示例：

```text
预算评审
```

### 4.3 触发搜索

用户可通过：

- 输入关键词后按 Enter。
- 点击搜索图标。
- 选择近期搜索。

### 4.4 清除搜索

搜索框有内容时展示清除按钮。

点击清除：

- 清空关键词。
- 搜索结果清空或回到当前邮件列表。
- 筛选项默认保留，避免误操作。

### 4.5 普通搜索边界

普通搜索不展示字段配置。

不要展示：

- 搜索内容：主题、正文、附件、发件人、收件人。
- 命中：主题。
- 命中：正文。
- 命中：附件。
- AI 分析。
- 保存搜索。

高级搜索未来可承载字段级配置。

## 5. 筛选栏

### 5.1 筛选定位

筛选栏属于搜索结果，不是导航。

规则：

- 紧贴结果摘要。
- 按高频到低频排列。
- 低频筛选进入 `更多筛选`。
- 启用筛选后展示筛选摘要。

### 5.2 筛选顺序

单账号场景：

1. 搜索范围
2. 时间
3. 发件人/收件人
4. 有附件
5. 排序
6. 更多筛选

多账号场景：

1. 账号
2. 搜索范围
3. 时间
4. 发件人/收件人
5. 有附件
6. 排序
7. 更多筛选

### 5.3 账号

仅多账号场景展示。

默认搜索当前账号，不默认搜索全部账号。

展示示例：

```text
账号：当前账号
账号：企业邮箱
账号：2 个账号
账号：全部账号
```

当搜索多个账号或全部账号时，每条结果必须展示来源账号。

### 5.4 搜索范围

展示：

```text
搜索范围：当前文件夹
```

可选项：

- 当前文件夹
- 全部邮件
- 收件箱
- 已发送
- 草稿
- 归档

规则：

- 用户从某个文件夹发起搜索时，默认搜索当前文件夹。
- 用户可切换到全部邮件。
- 如果多账号搜索，文件夹范围在所选账号内生效。

### 5.5 时间

展示：

```text
时间：全部时间
```

可选项：

- 全部时间
- 今天
- 昨天
- 本周
- 上周
- 本月
- 自定义时间范围

时间用于缩小结果范围，不把结果默认拆成大时间分组。

### 5.6 发件人/收件人

展示：

```text
人员：选择人员或团队
```

选择人员后可进一步指定角色：

- 任意相关人员
- 发件人
- 收件人
- 抄送人

人员展示优先使用：

```text
姓名 + 部门/职位 + 邮箱
```

不以头像作为主要识别信息。

### 5.7 有附件

展示：

```text
有附件：全部
```

可选项：

- 全部
- 有附件
- 无附件

### 5.8 排序

展示：

```text
排序：相关性优先
```

可选项：

- 相关性优先
- 时间从近到远
- 时间从远到近

默认排序：

- 优先相关性。
- 再按最近邮件优先。
- 未读、星标、近期往来可作为轻权重，不应压过强相关性。

### 5.9 更多筛选

更多筛选可包含：

- 未读状态。
- 星标状态。
- 是否有关联日程。
- 是否来自外部发件人。

启用后展示数量：

```text
更多筛选 · 2
```

## 6. 结果数量与摘要

### 6.1 默认展示

结果摘要展示：

```text
共找到 42 封邮件
清除筛选
```

如果需要补充摘要，可展示：

```text
未读 8
有附件 12
跨账号 2
```

摘要是辅助判断，不是主导航。

### 6.2 筛选摘要

启用筛选时展示短标签。

示例：

```text
已筛选：账号：企业邮箱 / 搜索范围：收件箱 / 有附件 / 发件人：王芳
```

规则：

- 不重复展示默认值。
- 清除筛选保留搜索关键词。
- 结果过多时提示缩小范围。

### 6.3 大量结果

企业邮箱结果可能达到上百或上千封。

规则：

- 默认展示有限数量。
- 每次继续显示固定数量。
- 提供显示全部，但不鼓励默认一次铺开。
- 超过 100 条时提示使用账号、时间、人员、附件缩小结果。
- 不使用无限滚动作为主要机制。

## 7. 结果列表

### 7.1 展示形式

默认使用高密度列表。

邮件搜索结果行应支持快速扫读：

- 谁发的。
- 什么主题。
- 哪个账号。
- 哪个文件夹。
- 什么时候。
- 是否未读、星标、有附件。
- 是否有关联日程。

### 7.2 列表与详情分栏

邮件搜索可以保留列表 + 详情分栏。

推荐：

- 左侧或中间为搜索结果列表。
- 右侧或主体为选中邮件详情。
- 选中邮件后在详情区阅读完整正文。
- 返回普通邮件列表时保留最近搜索关键词或状态。

### 7.3 默认数量

默认展示前 20 条。

规则：

- 继续显示：每次增加 20 条。
- 显示全部：用户明确选择后再展开。
- 不使用无限滚动。

### 7.4 选中与打开

交互：

- 单击选中邮件并在详情区打开。
- 双击可进入独立详情视图。
- Enter 打开或聚焦详情。
- 空格可切换选中或批量选择，视产品是否支持批量操作而定。

选中态不改变排序。

## 8. 单条邮件结果字段

### 8.1 必显字段

每条结果至少展示：

- 发件人
- 主题
- 时间

### 8.2 建议展示字段

存在时展示：

- 收件人或抄送摘要。
- 正文片段。
- 附件信息。
- 来源账号。
- 所在文件夹。
- 未读状态。
- 星标状态。
- 关联日程。

不存在时收起，不展示空值。

不要展示：

- 附件：0
- 正文：无
- 收件人：无
- 暂无附件

### 8.3 发件人

展示：

```text
王芳（财务部） <wangfang@company.com>
```

列表空间不足时可展示：

```text
王芳
```

悬停或详情中补全邮箱。

### 8.4 主题

展示邮件主题。

关键词出现在主题中时直接高亮。

规则：

- 保留 `Re:`、`Fw:` 等邮件上下文前缀。
- 主题为空时展示 `(无主题)`。
- 不在主题前放大图标。

### 8.5 正文片段

仅当正文与关键词相关时展示正文片段。

示例：

```text
正文：预算风险页已经更新，请在周五前确认……
```

不展示：

- 只由系统模板生成的摘要。
- 与关键词无关的正文。
- 大段正文。

### 8.6 附件信息

仅当邮件有附件时展示附件数量。

当附件名称与关键词相关时展示附件名称。

示例：

```text
附件 3
附件：Q4预算分配表.xlsx
附件：Q4预算分配表.xlsx 等 2 个
```

列表不提供打开附件按钮；附件操作进入邮件详情处理。

### 8.7 来源账号与文件夹

多账号搜索时展示来源账号。

示例：

```text
来源：企业邮箱｜gz123198@company.com
```

跨文件夹搜索时展示所在文件夹。

示例：

```text
文件夹：归档
```

单账号且当前文件夹内搜索时，可不重复展示来源账号和文件夹。

### 8.8 邮件状态

可展示：

- 未读
- 星标
- 已归档
- 草稿
- 已发送
- 外部发件人

状态标签必须短，不抢主题。

### 8.9 关联日程

当邮件关联日程时展示：

```text
关联日程
```

点击可打开日程详情。

如果关联日程已取消，可展示：

```text
关联日程已取消
```

不要把邮件搜索结果变成日程搜索结果；只展示必要线索。

## 9. 邮件详情区

### 9.1 详情内容

选中邮件后，详情区展示：

- 主题。
- 发件人。
- 收件人。
- 抄送。
- 时间。
- 来源账号。
- 所在文件夹。
- 正文。
- 附件。
- 关联日程。

详情区可以比列表更舒展。

### 9.2 详情操作

常用操作：

- 回复。
- 全部回复。
- 转发。
- 标为已读或未读。
- 星标或取消星标。
- 归档。
- 生成日程。
- 打开关联日程。

规则：

- 操作放在详情区顶部或工具栏中。
- 不在列表每行塞入全部操作。
- 危险或破坏性操作需要确认。

### 9.3 从邮件生成日程

当邮件需要转成日程时：

- 使用邮件主题作为日程标题默认值。
- 使用发件人、收件人、抄送推导参与人。
- 邮件正文进入日程描述。
- 邮件附件作为线索进入日程详情。
- 生成后回到日历创建任务或打开日程草稿。

## 10. 权限与隐私

### 10.1 多账号边界

多账号搜索必须避免混淆企业账号、项目账号和个人账号。

规则：

- 默认当前账号。
- 跨账号搜索需显式选择。
- 结果展示来源账号。
- 回复或转发时默认使用邮件来源账号。

### 10.2 共享邮箱

共享邮箱结果应展示共享邮箱名称或来源。

示例：

```text
来源：销售团队｜sales@company.com
```

发送、回复、归档等操作应遵守共享邮箱权限。

### 10.3 外部发件人

外部发件人可轻量标识。

示例：

```text
外部
```

不要用过重警告干扰普通检索，除非邮件存在安全风险。

## 11. 空状态

### 11.1 未输入关键词

未输入关键词时展示轻量空状态。

示例：

```text
搜索邮件
```

不要展示大面积说明或营销内容。

### 11.2 无结果

没有结果时展示：

```text
没有找到与“预算评审”相关的邮件
```

排查建议：

1. 扩大时间范围。
2. 改为搜索全部邮件。
3. 检查是否选择了正确账号。
4. 换用发件人、收件人、主题或附件名称搜索。
5. 清除筛选条件。

## 12. 响应式

### 12.1 宽屏

宽屏推荐：

- 左侧范围导航。
- 中间搜索结果列表。
- 右侧邮件详情。
- 筛选项横向排列。

### 12.2 窄屏

窄屏推荐：

- 搜索结果列表优先。
- 点击邮件进入详情。
- 详情页提供返回结果。
- 低频筛选进入更多筛选。
- 操作按钮换行但不遮挡主题。

### 12.3 不允许行为

避免：

- 搜索框位置跳层。
- 列表和详情同时挤到不可读。
- 附件图标或状态标签压过主题。
- 多账号结果不展示来源。
- 普通搜索展示复杂字段配置。

## 13. 验收清单

实现或评审邮件搜索结果页时，检查：

1. 搜索框是否保持在邮件模块顶部工具栏？
2. 普通搜索是否没有展示字段配置和命中来源？
3. 多账号场景是否默认当前账号？
4. 跨账号结果是否展示来源账号？
5. 筛选项是否按账号、搜索范围、时间、人员、附件、排序组织？
6. 结果摘要是否展示总数和已启用筛选？
7. 主体是否是高密度邮件列表，而不是大卡片？
8. 默认是否限制展示数量，并提供继续显示或显示全部？
9. 每条结果是否至少展示发件人、主题、时间？
10. 正文片段是否只在与关键词相关时展示？
11. 附件信息是否不展示空值？
12. 邮件详情区是否展示完整收发件和正文信息？
13. 列表行是否不塞入过多操作？
14. 从邮件生成日程是否保留邮件来源信息？
15. 无结果状态是否给出可执行排查建议？
16. 宽屏和窄屏是否都能稳定扫读？
