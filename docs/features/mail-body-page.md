# 邮件正文页功能规格

本文档定义 Coremail 邮箱模块中邮件正文 / 阅读窗格的产品规则、交互行为、组件边界和验收要求。更细的拆分文档保留在 `docs/mail-body-page/` 作为专项参考。

## 1. 页面定位

邮件正文页是企业邮箱中的高频工作处理页面，承担阅读、判断、处理、跟进和风险识别任务。

正文页不是单纯阅读页，而是企业邮件处理台。它必须帮助用户快速完成邮件处理闭环。

## 2. 核心任务

用户打开邮件正文页后，需要：

1. 判断邮件来源。
2. 判断邮件是否与自己相关。
3. 阅读正文内容。
4. 查看附件。
5. 判断是否存在安全风险。
6. 回复、回复全部、转发、归档、删除或移动邮件。
7. 将邮件转为待办、日程或跟进事项。
8. 处理完成后继续查看下一封邮件。

## 3. 页面结构

邮件正文页由以下区域组成：

1. 顶部操作区。
2. 邮件头部信息区。
3. 附件区。
4. 安全提示区。
5. 正文阅读区。
6. 历史引用区。
7. 快捷回复区。
8. 状态视图。

可选 AI 辅助能力只能作为正文处理后的辅助信息，不得放在附件、安全提示或正文之前，也不得替代发件人、收件关系、附件、安全和正文等核心判断字段。

## 4. 顶部操作区

默认展示：

- 回复。
- 回复全部。
- 转发。
- 归档。
- 删除。
- 更多。

可根据场景展示：

- 移动。
- 标记已读 / 未读。
- 添加旗标或跟进。
- 生成日程。
- 举报。

规则：

- 高频操作必须可见。
- 低频操作收进更多菜单。
- 滚动正文时操作区应保持可触达。
- Loading 或权限不足状态下禁用不可用操作。
- 删除和归档应有反馈，支持撤销时提供撤销入口。

## 5. 邮件头部信息区

必须展示：

- 主题。
- 发件人姓名。
- 发件人邮箱。
- 发送时间。
- 收件关系。

存在时展示：

- 外部发件人标识。
- 系统邮件标识。
- 重要性或旗标。
- 安全风险。
- 关联日程。

收件人规则：

- 默认展示摘要，例如 `发给我`、`抄送我`、`发给我等 6 人`。
- To、Cc、Bcc 明细可折叠。
- 外部收件人需要明确标识。
- 收件人较多时默认折叠并支持查看全部。

## 6. 时间规则

阅读窗格时间必须比列表更准确。

推荐：

- 今天：`今天 HH:mm`
- 昨天：`昨天 HH:mm`
- 本年：`M月D日 HH:mm`
- 跨年：`YYYY年M月D日 HH:mm`

如果当前页面已有统一短格式，可保持页面内一致，但不能在同一阅读窗格混用多套时间规则。

悬停时间可展示完整时间戳和时区。

## 7. 附件区

有附件时，附件区位于正文上方。

附件必须展示：

- 文件名。
- 文件类型。
- 文件大小。
- 安全扫描状态。
- 预览。
- 下载。

附件状态：

- 普通。
- 扫描中。
- 安全。
- 有风险。
- 已阻断。
- 不可用。

规则：

- 无附件时不展示附件区。
- 多附件超过 3 个时可展示前 3 个并提供查看全部。
- 扫描中禁用下载。
- 风险附件下载前二次确认。
- 高风险附件可阻断下载。

## 8. 安全提示

安全提示分为：

- 低风险：普通外部邮件，不阻断操作。
- 中风险：包含外部链接、陌生域名、敏感附件等，解释原因并对关键操作二次确认。
- 高风险：疑似钓鱼、病毒附件、伪造发件人等，可限制下载和跳转。

需要明确提示：

- 外部发件人。
- 外部收件人。
- 外部链接。
- 风险附件。
- 敏感邮件。
- 外部图片拦截。
- 安全拦截。

## 9. 正文阅读区

规则：

- 最新正文默认完整展示。
- 正文区域控制最大阅读宽度，避免长文本铺满页面。
- 图片自适应容器。
- 表格可横向滚动或适配容器。
- 链接需要可识别。
- 空正文展示 `此邮件没有正文内容`。

## 10. 历史引用区

历史引用默认折叠。

规则：

- 邮件链路超过 2 封时，仅默认展示最新正文。
- 用户点击 `展开历史邮件` 后查看完整引用。
- 签名和免责声明弱化展示。

## 11. 快捷回复

快捷回复默认轻量展示，点击后进入编辑。

规则：

- 发送中禁用重复提交。
- 系统通知类邮件可隐藏快捷回复区。
- 隐藏时展示 `此邮件为系统通知，不支持直接回复。`

## 12. 回复全部保护

回复全部涉及多人、外部人员、敏感邮件或风险邮件时，需要提示用户确认。

示例：

```text
本次回复将发送给多位收件人，其中包含外部人员。是否继续？
```

## 13. 已读规则

用户选中邮件后，不应在瞬间切换时立即标记已读。

推荐：

- 停留超过 2 秒后可标记已读。
- 滚动正文后可标记已读。
- 手动标记已读 / 未读优先。
- 快速切换邮件时不改变已读状态。

## 14. 连续处理

规则：

- 归档当前邮件后默认选中下一封。
- 删除当前邮件后默认选中下一封。
- 回复或转发后默认停留当前邮件。
- 标记、移动、添加跟进后默认停留当前邮件。

## 15. 状态与边界

正文页必须覆盖：

- 未选中邮件。
- 加载中。
- 正文加载失败。
- 附件加载失败。
- 邮件已删除。
- 权限不足。
- 安全拦截。
- 外部图片拦截。
- 空正文。
- 附件不可用。

## 16. 组件建议

建议拆分：

- `MailReadingPane`
- `MailActionBar`
- `MailHeader`
- `RecipientDetails`
- `SecurityNotice`
- `AttachmentList`
- `MailBody`
- `QuotedHistory`
- `QuickReply`
- `MailStateView`

Mock 数据不应写死在深层组件中，应放在 feature data 或 fixture 中。

## 17. 验收清单

实现或评审邮件正文页时，检查：

1. 主题、发件人姓名、发件人邮箱、发送时间和收件关系是否可见？
2. 外部发件人、系统邮件、重要邮件和安全风险是否有明确标识？
3. 回复、回复全部、转发、归档、删除和更多是否可用？
4. 附件文件名、类型、大小、预览、下载和安全状态是否可见？
5. 历史引用是否默认折叠？
6. 系统邮件是否可隐藏快捷回复并说明原因？
7. 回复全部包含外部人员时是否确认？
8. 删除或归档后是否有反馈和连续处理？
9. 是否覆盖未选中、加载、失败、权限不足、已删除、安全拦截、外部图片拦截和空正文状态？
