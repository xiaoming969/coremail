# Windows Outlook 日历提醒窗口优化

本文档定义日历提醒窗口的小步优化规则。范围严格收敛在现有 Outlook 类日历提醒能力：默认提醒时间、单个日程提醒时间、不提醒、提醒弹窗和提醒窗口基础操作。不做智能提醒、不做复杂分级、不做跨设备入会判断。

上游依据：

- `docs/product.md`
- `docs/design.md`
- `docs/information-architecture.md`
- `docs/components.md`

## 0. PRD 结论

需求名称：`Windows Outlook 日历提醒窗口优化`。

需求目标：

```text
基于现有 Outlook 日历提醒能力，优化提醒弹窗的触发规则和基础设置能力。已结束会议不再提醒，正在进行中的会议仍可提醒；用户可设置自己新建日程的默认提醒时间，也可设置收到会议邀请时的提醒策略；用户点击加入会议后，该会议在当前客户端不再重复提醒。
```

核心规则：

```text
1. 全局关闭提醒弹窗时，不展示任何日历提醒弹窗。
2. 单个日程设置为“不提醒”时，该日程不展示提醒弹窗。
3. 未来会议到达提醒时间后，正常展示提醒。
4. 正在进行中的会议仍可展示提醒。
5. 已结束会议不展示提醒。
6. 用户在当前客户端点击“加入会议”后，该日程实例后续不再重复提醒。
7. 新建日程默认提醒时间支持“不提醒、开始时、5 分钟前、10 分钟前、15 分钟前、30 分钟前、1 小时前、1 天前”。
8. 收到会议邀请时默认跟随组织者设置；用户可明确覆盖为固定提醒时间或不提醒。
9. 单个日程的提醒设置优先级最高。
```

## 1. 需求定位

用户反馈集中在提醒弹窗的基础有效性：

- 已结束会议继续提醒会造成无效打扰。
- 正在进行中的会议仍有加入价值，不能按开始时间过期直接取消提醒。
- 用户点击加入会议后，不希望同一日程在当前客户端继续重复提醒。
- 默认 15 分钟提醒不适合所有新建日程，需要 5 分钟、10 分钟、30 分钟或不提醒。
- 收到会议邀请时需要区分“跟随组织者设置”和“用户明确覆盖提醒策略”。
- 用户需要在设置中统一控制日历提醒弹窗、新建日程默认提醒时间和收到邀请时的提醒策略。

## 2. 设计目标

- 已结束会议不再弹提醒。
- 正在进行中的会议仍可提醒。
- 用户在当前客户端点击“加入会议”后，该日程实例后续不再重复提醒。
- 支持修改新建日程默认提醒时间，默认值为 `15 分钟前`。
- 支持设置收到会议邀请时的提醒策略，默认值为 `跟随组织者设置`。
- 支持全局关闭日历提醒弹窗。

## 3. 本期范围

本期包含：

- 设置页增加 `日历提醒` 模块。
- 支持开启 / 关闭日历提醒弹窗。
- 支持设置新建日程默认提醒时间。
- 支持设置收到会议邀请时的提醒策略：跟随组织者设置、不提醒或固定提前量。
- 新建日程默认提醒时间可设置为 `不提醒`。
- 未来会议到达提醒时间后正常提醒。
- 正在进行中的会议仍可提醒。
- 已结束会议不再提醒。
- 用户点击“加入会议”后，本客户端不再重复提醒该日程实例。
- 单个日程设置为 `不提醒` 时不弹窗。
- 提醒窗口展示 `即将开始` / `正在进行中` 状态。
- 保留加入会议、双击查看详情、内联稍后提醒、关闭操作。
- 提醒弹窗提供轻量入口，引导用户进入提醒设置调整默认提醒时间。

本期不包含：

- 不做会议重要程度智能判断。
- 不按领导、共享日历、会议类型分别设置提醒规则。
- 不根据线上 / 线下、地点距离自动调整提醒时间。
- 不判断用户是否真的进入 Teams、Zoom 或第三方会议。
- 不做跨设备同步“已点击加入会议”状态。
- 不做复杂提醒聚合、优先级排序、AI 推荐和规则模板。

## 5. 设置项

### 5.1 开启日历提醒弹窗

控件类型：开关。默认开启。

规则：

- 开启后，日程到达提醒时间时显示提醒弹窗。
- 关闭后，所有日历提醒弹窗不再显示。
- 关闭开关不影响日程本身的提醒字段。
- 关闭开关不删除已有日程的提醒时间。
- 重新开启后，仅后续符合条件的提醒继续生效。
- 关闭期间已结束的会议，重新开启后不补弹。

状态结果：

| 开关状态 | 结果 |
| --- | --- |
| 开启 | 按提醒规则弹窗 |
| 关闭 | 不显示提醒弹窗 |
| 关闭后重新开启 | 仅对后续符合条件的提醒生效 |

### 5.2 新建日程的默认提醒时间

控件类型：下拉选择。默认值：`15 分钟前`。

选项：

```text
不提醒
开始时
5 分钟前
10 分钟前
15 分钟前
30 分钟前
1 小时前
1 天前
```

规则：

- 只影响用户自己新建的日程。
- 已创建日程是否跟随变化，沿用现有逻辑，本期不额外改造。
- 单个日程提醒时间优先级高于新建日程默认提醒时间。
- 默认设置为 `不提醒` 时，新建日程默认不设置提醒。
- 新建日程默认提醒时间只控制提前量，不控制弹窗是否出现；弹窗仍受全局开关控制。

示例：

| 用户设置 | 新建日程默认提醒 |
| --- | --- |
| 15 分钟前 | 新建日程默认提前 15 分钟提醒 |
| 5 分钟前 | 新建日程默认提前 5 分钟提醒 |
| 不提醒 | 新建日程默认不提醒 |

### 5.3 收到会议邀请时的提醒时间

控件类型：下拉选择。默认值：`跟随组织者设置`。

选项：

```text
跟随组织者设置
不提醒
开始时
5 分钟前
10 分钟前
15 分钟前
30 分钟前
1 小时前
1 天前
```

规则：

- 收到会议邀请后，系统根据当前用户的提醒策略生成该用户本地的最终提醒时间。
- 如果用户选择 `跟随组织者设置`，则按组织者发送的提醒时间提醒；如果组织者未设置提醒，则该日程不提醒。
- 如果用户选择具体提醒时间，则收到的会议默认按用户自己的提醒时间提醒，不再按组织者设置执行。
- 如果用户选择 `不提醒`，则收到的会议默认不进入提醒弹窗。
- 用户仍可在单个日程详情页修改提醒时间，单个日程设置优先级最高，仅对当前用户生效，不影响组织者和其他参会人。

优先级：

```text
单个日程用户手动修改的提醒时间
>
用户在设置中选择的“收到会议邀请时的提醒时间”
>
组织者发送的提醒时间
>
无提醒
```

## 6. 新建 / 编辑日程提醒设置

字段名称：`提醒`。控件类型：下拉选择。

规则：

- 新建日程时，提醒字段默认读取设置页的 `新建日程的默认提醒时间`。
- 用户可以在单个日程中修改提醒时间。
- 单个日程修改后，该日程以后按单独设置执行。
- 单个日程设置为 `不提醒` 后，该日程不弹出提醒。
- 周期日程修改提醒时，沿用当前 Outlook 类处理方式：用户选择修改单次或整个系列。
- 编辑已有日程时，展示该日程当前提醒时间，不自动改为最新默认提醒时间。

## 7. 提醒窗口字段

提醒窗口包含：

- 日程标题：必须展示，过长可截断。
- 会议时间：必须展示开始时间，建议展示结束时间。
- 会议状态：`即将开始` 或 `正在进行中`。
- 地点 / 会议室：存在时展示，不存在时收起。
- 加入会议入口：存在可识别线上会议链接且会议未结束时，在提醒条目右侧操作区展示。
- 所属日历：建议展示，用于判断自己的日历、共享日历或领导日历。
- 多个提醒采用紧凑列表行展示，避免单条提醒占用过高空间；加入会议、稍后提醒、清除等高频操作保持在每条提醒右侧。

已结束会议不展示提醒窗口，因此不展示 `已结束` 状态。

## 8. 提醒窗口操作

### 8.1 加入会议

展示条件：

- 日程存在可识别线上会议链接。
- 当前状态为 `即将开始` 或 `正在进行中`。
- 会议未结束。

点击后：

- 打开会议链接。
- 当前客户端记录该日程实例为 `已点击加入会议`。
- 该日程实例后续不再重复弹出提醒。
- 不影响其他日程提醒。
- 不做跨设备同步。
- 不判断用户是否实际进入会议。

准确表述为：用户在当前客户端点击“加入会议”后，该日程实例在当前客户端不再重复提醒。

### 8.2 查看详情

- 不作为主按钮展示。
- 用户双击提醒条目后查看日程详情页。
- 不自动标记为已处理。
- 不等同于已加入会议。

### 8.3 稍后提醒

提醒弹窗里的稍后提醒以内联下拉按钮展示，用于推迟本次弹窗，不修改日程自身的默认提醒时间。

稍后提醒只表示从当前时间起推迟：

```text
5 分钟后
10 分钟后
15 分钟后
30 分钟后
1 小时后
```

规则：

- 当前提醒暂时关闭。
- 到达稍后提醒时间后重新判断日程状态。
- 不提供 2 小时、1 天、1 周等长时段选项；长时段更像修改提醒规则，应进入日程详情处理。
- 如果会议仍未结束，可再次提醒。
- 如果会议已经结束，不再提醒。
- 如果用户在稍后期间点击过 `加入会议`，不再提醒。

### 8.4 清除

- 以每条提醒右侧的轻量关闭按钮展示。
- 和 `加入会议`、`稍后提醒` 保持同一行，但通过弱化颜色和尺寸降低操作权重。
- 清除对应单条提醒。
- 清除后如果提醒队列仍有其他日程，焦点移动到下一条提醒。
- 清除后如果提醒队列为空，关闭提醒窗口。
- 同一提醒时间不重复弹出。
- 不取消日程。
- 周期会议默认只关闭当前实例提醒，除非现有逻辑另有规则。

### 8.5 全部清除

- 当前存在提醒时可用。
- 清除当前提醒窗口中的全部提醒。
- 清除后关闭提醒窗口。
- 不取消任何日程。
- 不改变日程本身的提醒字段。

## 9. 提醒触发优先级

提醒触发时按以下顺序判断：

```text
1. 全局提醒弹窗是否开启
2. 当前用户对该日程的最终提醒时间是否为不提醒
3. 日程是否已取消
4. 当前时间是否已晚于会议结束时间
5. 当前客户端是否已点击过加入会议
6. 当前时间是否到达提醒时间
7. 是否展示提醒窗口
```

完整判断流程：

```text
开始
↓
是否开启日历提醒弹窗？
├── 否：不弹窗
└── 是
    ↓
当前用户对该日程的最终提醒时间是否为“不提醒”？
    ├── 是：不弹窗
    └── 否
        ↓
该日程是否已取消？
        ├── 是：不弹窗
        └── 否
            ↓
当前时间是否晚于会议结束时间？
            ├── 是：不弹窗
            └── 否
                ↓
用户是否已在当前客户端点击过“加入会议”？
                ├── 是：不弹窗
                └── 否
                    ↓
当前时间是否到达提醒时间？
                    ├── 否：不弹窗，等待提醒时间
                    └── 是：弹出提醒窗口
```

## 10. 时间状态判断

普通会议：

```text
当前时间 < 开始时间：
即将开始

开始时间 <= 当前时间 <= 结束时间：
正在进行中

当前时间 > 结束时间：
已结束，不提醒
```

没有结束时间的会议：沿用系统默认时长或现有逻辑。

全天日程：本期沿用现有提醒逻辑；如果必须判断，日期已结束后不再提醒。

跨天日程：按开始时间和结束时间判断，当前时间在两者之间视为正在进行中。

周期日程：按当前实例判断，不按整个系列判断。

## 11. 客户端启动 / 唤醒补弹规则

当 Windows Outlook 启动、恢复运行或网络恢复后，检查未处理提醒：

- 已结束：不补弹。
- 正在进行中：可以补弹。
- 未开始且已到提醒时间：可以补弹。
- 未到提醒时间：不弹，等待提醒时间到达。

## 12. 已点击加入会议规则

记录范围：

```text
记录范围：当前客户端
记录对象：当前日程实例
记录行为：点击“加入会议”
记录结果：该日程实例后续不再提醒
```

不记录为已加入的情况：

- 复制会议链接。
- 查看日程详情。
- 从浏览器历史进入会议。
- 从 Teams、手机端、他人转发链接进入会议。
- 打开会议链接但页面加载失败。

周期会议只标记当前实例，不影响后续周期实例。

## 13. 弹窗状态文案

未来会议：

```text
X 分钟后开始
```

正在进行中：

```text
已开始 X 分钟
```

已结束：

```text
不展示提醒弹窗
```

## 14. 多个提醒同时触发

本期不做复杂聚合改造，沿用提醒窗口列表方式。

过滤规则：

- 先过滤已结束会议。
- 再过滤已取消会议。
- 再过滤已点击加入会议的会议。
- 再展示剩余提醒。

排序规则：按会议开始时间从早到晚排序。

## 15. 异常与边界

- 会议时间修改后，提醒时间按新的会议开始时间重新计算。
- 修改后会议已经结束：不提醒。
- 会议被取消：不展示提醒弹窗。
- 用户修改单个日程提醒时间后：该日程按新提醒时间执行。
- 全局提醒关闭后又开启：已结束不补弹，仍在进行中可提醒，未来会议到达提醒时间后正常提醒。
- 稍后提醒到期时会议已结束：不再弹窗。
- 稍后提醒期间点击加入会议：到期不再弹窗。
- 电脑休眠、锁屏、离线恢复后：按统一规则重新判断。
- 跨时区会议：按当前已换算后的本地时间判断。

## 16. 权限和共享日历

- 共享日历提醒沿用当前逻辑。
- 不新增按共享日历、领导日历、会议室日历单独设置提醒。
- 私密日程提醒沿用当前逻辑，按用户可见字段展示。
- 不新增私密日程特殊提醒规则、权限申请入口或内容透出。

## 18. 数据状态

日程实例可判断以下提醒状态：

```text
未到提醒时间
待提醒
已提醒
已关闭
已稍后提醒
已点击加入会议
已结束
已取消
不提醒
```

| 状态 | 含义 | 是否弹窗 |
| --- | --- | --- |
| 未到提醒时间 | 当前时间未到提醒触发点 | 否 |
| 待提醒 | 已到提醒时间，符合提醒条件 | 是 |
| 已提醒 | 已展示过提醒 | 根据现有逻辑 |
| 已关闭 | 用户关闭了提醒 | 否 |
| 已稍后提醒 | 用户选择稍后提醒 | 到时重新判断 |
| 已点击加入会议 | 当前客户端点击过加入会议 | 否 |
| 已结束 | 当前时间晚于结束时间 | 否 |
| 已取消 | 会议已取消 | 否 |
| 不提醒 | 当前用户最终提醒时间为不提醒 | 否 |

## 19. 验收标准

- 会议 `10:00 - 11:00`，当前 `12:00`：不弹窗。
- 会议 `10:00 - 11:00`，当前 `10:10`：弹窗，状态为 `正在进行中 / 已开始 10 分钟`。
- 会议 `10:00 - 11:00`，当前 `09:45` 且提醒为 `15 分钟前`：弹窗，状态为 `即将开始`。
- 用户点击 `加入会议` 后，同一日程实例不再重复弹窗。
- 全局 `开启日历提醒弹窗 = 关闭` 时不弹窗。
- 新建日程默认提醒时间改为 `5 分钟前` 后，新建日程默认使用 `5 分钟前`。
- 收到会议邀请时的提醒时间为 `跟随组织者设置` 时，按组织者提醒时间生成当前用户本地提醒。
- 收到会议邀请时的提醒时间改为 `不提醒` 后，收到的会议默认不进入提醒弹窗。
- 单个日程设置为 `不提醒` 时不弹窗。
- 稍后提醒到期时会议已结束：不弹窗。

## 20. 最终核心规则

```text
1. 全局关闭提醒弹窗时，不展示任何日历提醒弹窗。
2. 当前用户最终提醒时间为“不提醒”时，该日程不展示提醒弹窗。
3. 未来会议到达提醒时间后，正常展示提醒。
4. 正在进行中的会议仍可展示提醒。
5. 已结束会议不展示提醒。
6. 用户在当前客户端点击“加入会议”后，该日程实例后续不再重复提醒。
7. 新建日程默认提醒时间支持“不提醒、开始时、5 分钟前、10 分钟前、15 分钟前、30 分钟前、1 小时前、1 天前”。
8. 收到会议邀请时默认跟随组织者设置；用户可明确覆盖为固定提醒时间或不提醒。
9. 单个日程的提醒设置优先级最高。
```
