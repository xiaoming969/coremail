# Coremail 信息架构

本文档定义 Coremail 的模块边界、页面层级、导航关系和主要用户路径。视觉和组件细节见 `docs/design.md`、`docs/components.md` 和 `docs/patterns/`。

## 1. 架构原则

### 1.1 一级模块稳定

Coremail 的一级模块：

- 邮箱。
- 日历。
- 通讯录。
- 设置。

搜索、详情、创建、提醒、共享权限、邮件正文、人员选择都是模块内部状态或任务层，不是一级模块。

### 1.2 左侧定范围，顶部定任务，主体做判断

全局布局职责：

- 左侧：账号、邮箱文件夹、日历范围、组织范围、模块切换。
- 顶部：搜索、同步、提醒、视图切换、当前页面高频工具。
- 主体：邮件列表、阅读窗格、日历画布、搜索结果、联系人列表、详情内容。
- 浮层：创建、编辑、权限、确认、更多信息。

不要把同一类入口分散到多个区域，也不要让搜索后控件突然跳到另一个层级。

### 1.3 搜索是状态，不是新产品

搜索属于当前模块的检索状态。

- 邮件搜索保留在邮箱模块。
- 日程搜索保留在日历模块。
- 人员搜索保留在通讯录模块或人员选择任务中。
- 全局搜索也必须展示来源并保留返回路径。

搜索结果页不应表现成独立 AI 产品或推荐页。

### 1.4 任务浮层保持来源上下文

任务浮层包括：

- 写邮件。
- 新建 / 编辑日程。
- 邮件详情或阅读窗格。
- 日程详情。
- 联系人详情。
- 共享与权限。
- 查看权限。
- 添加共享日历。
- 确认框。

浮层关闭后必须回到打开它的来源上下文。

## 2. 一级模块

### 2.1 邮箱

邮箱模块用于处理邮件、附件、会话、文件夹、搜索和阅读。

左侧：

- 收藏夹。
- 邮箱账号。
- 收件箱。
- 未读邮件。
- 旗标邮件。
- 草稿箱。
- 已发送。
- 已归档。
- 垃圾邮件。
- 已删除。
- 文件夹 / 标签。
- 底部产品切换。

顶部：

- 邮件搜索。
- 刷新。
- 多选。
- 筛选。
- 当前邮件或阅读窗格工具。

主体：

- 邮件列表。
- 邮件搜索结果。
- 邮件正文 / 阅读窗格。
- 附件处理。
- 安全提示。
- 写邮件任务。

邮箱模块不承载日程搜索结果；涉及日程检索时进入日历模块。邮件中的关联日程可打开日程详情或创建日程任务。

### 2.2 日历

日历模块用于查看、创建、搜索、提醒和管理日历权限。

左侧：

- 我的日历。
- 共享日历。
- 群组日历。
- 会议室日历。
- 添加共享日历入口。
- 日历项菜单。
- 底部产品切换。

顶部：

- 日程搜索。
- 同步日历。
- 提醒。
- 今天。
- 上一段 / 下一段。
- 月 / 周 / 日 / 列表视图。
- 叠加视图。

主体：

- 月视图。
- 周视图。
- 日视图。
- 列表视图。
- 日程搜索结果。
- 日程详情。
- 新建 / 编辑日程。
- 会议邀请。
- 提醒弹窗。
- 权限管理。

日历搜索属于日历模块内状态，不应替代完整日历视图。

### 2.3 通讯录

通讯录模块用于查找人员、理解组织关系、确认联系方式和选择协作对象。

左侧：

- 我的联系人。
- 企业通讯录。
- 组织架构。
- 群组 / 部门。
- 外部联系人。
- 最近联系人。

顶部：

- 人员搜索。
- 筛选。
- 新建联系人或添加外部联系人。

主体：

- 联系人列表。
- 联系人详情。
- 组织架构树。
- 人员搜索结果。
- 人员选择。
- 群组 / 部门详情。

通讯录中的人员选择可以被邮箱、日历、权限共享任务复用，但仍应保持人员对象语义清楚。

### 2.4 设置

设置模块承载低频偏好和系统配置。

可包含：

- 账号设置。
- 邮件设置。
- 日历设置。
- 通知偏好。
- 日历提醒。
- 默认日历。
- 安全与隐私。
- 系统状态。

日常共享权限操作仍优先保留在日历或邮箱上下文中，不应过早迁入设置。

## 3. 全局能力

### 3.1 搜索

搜索包括：

- 邮件搜索。
- 日程搜索。
- 通讯录搜索。
- 全局搜索。

全局搜索必须展示结果来源，例如邮箱、日历、联系人、文件夹、日历来源或账号来源。

搜索结果必须保留返回路径，不能让用户失去当前模块上下文。

### 3.2 多账号 / 多来源

多账号和多来源必须明确展示：

- 邮箱账号。
- 邮件文件夹。
- 日历来源。
- 共享来源。
- 群组或会议室来源。
- 联系人组织来源。

不要把账号来源、日历来源、组织来源混写成一个泛化“来源”字段。

### 3.3 权限

权限能力包括：

- 共享日历权限。
- 共享邮箱 / 委托邮箱权限。
- 联系人可见范围。
- 邮件正文和附件访问权限。
- 日程可见范围。

权限不足时必须说明状态和可行下一步。

### 3.4 安全

安全能力包括：

- 外部发件人。
- 外部收件人。
- 外部链接。
- 风险附件。
- 外部图片拦截。
- 敏感邮件。
- 被阻断内容。

安全提示必须靠近风险对象展示。

### 3.5 通知

通知包括：

- 日历提醒。
- 邮件处理反馈。
- 权限变更反馈。
- 共享邀请。
- 操作成功或失败。

通知不替代主路径；它必须能回到具体邮件、日程、联系人或权限任务。

### 3.6 空状态 / 错误状态

所有模块必须覆盖：

- 加载中。
- 空状态。
- 搜索无结果。
- 错误状态。
- 权限不足。
- 安全风险。
- 数据不可用。

空状态说明原因和下一步，不做营销介绍。

## 4. 主要用户路径

### 4.1 处理邮件

1. 进入邮箱模块。
2. 在左侧选择账号或文件夹。
3. 在邮件列表中扫读邮件。
4. 打开阅读窗格。
5. 判断发件人、主题、时间、附件和安全状态。
6. 回复、回复全部、转发、归档、删除或移动。
7. 处理后继续下一封。

### 4.2 查找邮件或资料

1. 进入邮箱模块。
2. 在顶部搜索框输入关键词。
3. 进入邮件搜索结果。
4. 通过发件人、主题、摘要、时间、附件和文件夹判断目标邮件。
5. 打开邮件详情或阅读窗格处理。

### 4.3 查看今天安排

1. 进入日历模块。
2. 选择月、周、日或列表视图。
3. 查看当前日程。
4. 打开日程详情或加入会议。

### 4.4 查找会议

1. 进入日历模块。
2. 在顶部搜索框输入关键词。
3. 进入日程搜索结果。
4. 通过时间、组织者、参与人、地点和来源判断目标日程。
5. 打开详情。
6. 关闭后回到搜索结果。

### 4.5 查找联系人

1. 进入通讯录模块。
2. 搜索姓名、部门、职位、邮箱或手机号。
3. 在结果中通过部门、职位、邮箱区分人员。
4. 打开联系人详情或选择人员。

### 4.6 管理共享权限

1. 在日历或邮箱上下文找到可共享对象。
2. 打开共享与权限。
3. 查看对象、成员、权限和状态。
4. 添加、修改或取消共享。
5. 操作后回到原上下文。

## 5. 命名规则

- `模块`：一级产品区域，例如邮箱、日历、通讯录、设置。
- `页面状态`：模块内部主体状态，例如列表、搜索结果、详情、空状态。
- `任务浮层`：短期任务容器，例如写邮件、新建日程、权限面板。
- `来源`：数据来自哪个账号、文件夹、日历、组织或共享对象。
- `权限`：当前用户对对象的可见和可操作范围。

## 6. 信息架构检查清单

新增或修改页面前，检查：

1. 它属于哪个一级模块？
2. 它是主工作区状态，还是任务浮层？
3. 它是否改变搜索、详情、创建或返回路径？
4. 它是否让用户失去当前账号、文件夹、日历或组织上下文？
5. 它是否混淆发件人、收件人、组织者、参与人、账号来源、日历来源或组织来源？
6. 它是否把权限操作迁移到了错误层级？
7. 它是否能在大量企业数据下保持可扫读？
