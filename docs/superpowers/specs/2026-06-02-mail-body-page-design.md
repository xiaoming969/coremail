# 邮件正文页优化设计

## 背景

当前项目是 React 19、Vite、Tailwind 4 的 Coremail 演示前端。邮件模块已有列表、写信弹窗、简单阅读区、AI 摘要、删除撤销和关联日程入口，但阅读区正文、附件、安全、收件人和异常状态主要集中在 `src/App.jsx` 的内联 JSX 中。

## 目标

在不重写邮件列表、全局路由、构建配置和无关页面的前提下，增强右侧邮件阅读窗格，使它覆盖企业邮箱正文页的核心信息层级、处理动作、安全感知、异常状态和基础可访问性。

## 方案

采用现有分栏结构，保留 `MailWorkspace` 作为邮件工作台容器，把正文页视图拆到 `src/features/mail/components/MailReadingPane.tsx`。阅读窗格接收当前选中邮件、关联日程、操作回调和本地演示状态，通过小组件组合渲染操作栏、头部、收件人、附件、安全提示、正文、历史引用、快捷回复和状态页。

## 数据

新增 `src/features/mail/types/mail.ts` 定义企业邮件正文页类型。新增 `src/features/mail/data/mockMailReadingStates.ts` 提供 8 类正文页演示状态与状态触发辅助函数。现有 `MOCK_MAILS` 继续作为列表数据源，并扩展部分邮件字段以覆盖外部邮件、风险附件、长邮件链、系统通知、权限不足、已删除、加载失败、安全拦截、外部图片拦截等场景。

## 交互

回复、回复全部、转发继续复用现有写信弹窗。回复全部在外部人员或多人场景下先弹本地确认。归档、删除继续使用现有 toast 和撤销能力，并在操作后选中下一封。标记、跟进、附件下载、附件预览和快捷回复通过本地 toast 模拟。已读规则从选中即标记调整为停留 2 秒或滚动后标记，手动标记优先。

## 风险

当前项目没有真实接口、权限系统、附件预览服务、病毒扫描服务和富文本 sanitizer。实现以 mock 数据和安全文本渲染为主；HTML 正文不使用 `dangerouslySetInnerHTML`，避免引入新依赖。
