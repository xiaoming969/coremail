# Coremail 组件规范

本文档定义 Coremail 演示产品中基础组件的使用规则。它不追求覆盖所有视觉 token，而是约束组件语义、信息密度、交互行为和禁止用法。

组件实现应同时遵守：

- `docs/product.md`
- `docs/design.md`
- `docs/information-architecture.md`
- 具体功能文档，例如 `docs/features/shared-permission.md`

## 1. 组件设计原则

### 1.1 组件服务任务，不服务装饰

Coremail 是企业办公工具。组件应帮助用户快速判断、筛选、查看和执行操作。

优先：

- 清晰的信息层级
- 稳定的位置和尺寸
- 可扫读的密度
- 明确的操作后果

避免：

- 装饰性大卡片
- 过度圆角和大面积留白
- 与任务无关的图标
- 用组件解释系统逻辑

### 1.2 语义先于样式

组件样式必须表达真实语义。

示例：

- `从列表移除` 不能写成 `删除`。
- 颜色分类标签不能替代会议状态。
- 组织者和参与人不能合并成同一个“人员”字段。
- 来源账号和日历来源不能混写。

### 1.3 同类组件行为一致

同一类组件在不同页面中应保持一致。

示例：

- hover 菜单和右键菜单内容保持一致。
- 搜索框清除按钮位置保持一致。
- 详情弹窗关闭后回到打开来源。
- 筛选项启用后都能通过“清除筛选”恢复。

### 1.4 基础字号与尺寸

组件不得自行创造游离字号。默认遵守 `docs/design.md` 的字号阶梯：

- 12px：标签、数量徽标、辅助说明、低权重状态。
- 14px：筛选器、按钮、列表 meta 信息、普通正文。
- 16px：列表标题、关键字段、重要操作文本。
- 24px：详情页标题、页面级标题。
- 32px：极少使用，仅用于真正一级工作台标题。

规则：

- 日程搜索列表只使用 12px、14px、16px。
- 筛选器高度建议 36px。
- 列表按钮高度建议 32px 或 36px。
- 组件尺寸变化必须服务信息层级或操作效率，不能只为制造视觉差异。

## 2. 按钮

### 2.1 按钮类型

Coremail 常用按钮类型：

| 类型 | 用途 | 示例 |
| --- | --- | --- |
| 主按钮 | 当前任务的主要完成动作 | 保存、确认共享、一键入会 |
| 次按钮 | 常规动作或次要路径 | 查看详情、取消 |
| 文本按钮 | 低干扰辅助动作 | 清除筛选、显示全部 |
| 图标按钮 | 高频工具或空间受限操作 | 搜索、关闭、上一段、下一段 |
| 危险按钮 | 会造成删除、撤销或不可逆影响 | 删除日历、取消共享 |

### 2.2 主按钮

主按钮只用于当前界面最重要的动作。

规则：

- 一个任务区域中原则上只有一个主按钮。
- 文案必须是动词或动宾结构。
- 不能用模糊文案，例如 `确定`，除非确认对象在标题中已经非常明确。

推荐：

- 保存
- 确认共享
- 一键入会
- 从列表移除

不推荐：

- 确定
- 提交
- 删除共享日历
- 好的

### 2.3 次按钮

次按钮用于保留路径或辅助操作。

规则：

- 不抢主按钮视觉权重。
- 在列表中，`查看详情` 通常是次按钮。
- 当没有主动作时，次按钮可以是唯一按钮。

### 2.4 图标按钮

图标按钮用于空间有限且图标含义明确的场景。

适用：

- 搜索
- 清除
- 关闭
- 上一段
- 下一段
- 更多菜单
- 同步
- 提醒

规则：

- 必须提供 `aria-label` 或 `title`。
- 不熟悉的图标需要 tooltip。
- 不用自绘图标替代已有图标库中的常见图标。
- 不用图标承载关键业务语义，除非旁边有文字。

### 2.5 危险按钮

危险按钮用于删除、取消共享、移除等可能造成影响的操作。

规则：

- 危险动作进入确认框。
- 确认框必须说明影响范围。
- 主按钮文案必须表达真实后果。

示例：

- 删除日历：删除我拥有的日历。
- 取消共享：撤销某个对象的访问权限。
- 从列表移除：只从我的共享日历列表移除，不影响共享方设置。

## 3. 搜索框

### 3.1 位置

搜索框属于顶部工具栏。

规则：

- 搜索前后位置保持一致。
- 搜索结果页不再另起一个主体搜索框。
- 进入搜索状态时可显示 `返回日历`。

### 3.2 尺寸

搜索框默认保持工具栏尺寸，不因进入搜索结果页突然变成长条。

规则：

- 工具栏搜索框使用 14px 输入文字。
- 搜索关键词较重要时可使用 16px，但搜索前后必须一致。
- 普通搜索框不使用页面标题级字号。
- 搜索结果页不得新增第二个主体大搜索框。

可以使用扩展搜索弹层承载：

- 近期搜索
- 高级搜索入口
- 搜索建议

但弹层关闭后，搜索框仍回到工具栏尺寸。

### 3.3 清除按钮

搜索框有内容时展示清除按钮。

点击清除：

- 清空关键词。
- 搜索结果清空或回到默认状态。
- 筛选项默认保留，避免误操作。

### 3.4 普通搜索边界

普通搜索框不展示字段配置。

不要在普通搜索结果页展示：

- 搜索内容：标题、正文、附件
- 命中来源
- 保存搜索
- AI 搜索说明

这些能力应放到高级搜索或独立能力中评估。

## 4. 筛选器

### 4.1 筛选器定位

筛选器属于当前列表或搜索结果，不是导航。

规则：

- 筛选栏靠近被筛选内容。
- 筛选项按业务优先级排列。
- 筛选结果通过摘要反馈。
- 筛选项不能变成页面标题。

### 4.2 日程搜索筛选顺序

单账号场景：

1. 搜索范围
2. 时间
3. 颜色分类
4. 排序
5. 更多筛选

多账号场景：

1. 账号
2. 搜索范围
3. 时间
4. 颜色分类
5. 排序
6. 更多筛选

更多筛选中可放：

- 人员
- 会议类型
- 有附件

### 4.3 筛选控件展示

筛选控件使用“名称：当前值”的格式。

示例：

- 搜索范围：全部日历
- 时间：全部时间
- 颜色分类：全部
- 排序：相关性优先
- 账号：当前账号

规则：

- 名称不能省略。
- 当前值必须可扫读。
- 低频筛选可以折叠到“更多筛选”。
- 启用低频筛选后，“更多筛选”应显示数量。

### 4.4 清除筛选

清除筛选用于恢复默认筛选条件。

规则：

- 保留搜索关键词。
- 清除筛选条件。
- 刷新结果列表。
- 当没有启用筛选时，可降低视觉权重但不必隐藏。

## 5. 菜单

### 5.1 菜单类型

Coremail 使用两类菜单：

- 更多菜单：通过 `...` 或下拉按钮打开。
- 右键菜单：桌面端效率增强。

右键菜单不是唯一入口。

### 5.2 内容一致

同一对象的 hover 更多菜单和右键菜单内容应一致。

示例：

我的日历菜单：

1. 颜色
2. 重命名
3. 共享与权限
4. 导出
5. 删除日历

共享日历菜单：

1. 颜色
2. 查看权限
3. 从列表移除

### 5.3 菜单排序

菜单项按任务风险和频率排序：

1. 高频低风险操作。
2. 查看或配置类操作。
3. 导出等低频操作。
4. 危险操作。

危险操作放在末尾，并使用危险色或分隔线。

### 5.4 菜单项文案

菜单项使用短动词或动宾结构。

推荐：

- 查看权限
- 共享与权限
- 从列表移除
- 删除日历

不推荐：

- 设置
- 管理
- 更多
- 删除共享日历

## 6. 弹窗与确认框

### 6.1 弹窗用途

弹窗用于短流程任务。

适用：

- 共享与权限
- 查看共享权限
- 添加共享日历
- 从列表移除确认
- 提醒
- 快捷键帮助

不适用：

- 承载复杂导航
- 展示大量搜索结果
- 替代主页面

### 6.2 结构

标准弹窗结构：

1. 标题
2. 简短说明
3. 主体内容
4. 底部操作

规则：

- 标题说明任务对象。
- 说明文字只解释必要后果。
- 主按钮放在底部右侧。
- 关闭后回到来源上下文。

### 6.3 确认框

确认框用于确认有风险或有后果的动作。

必须包含：

- 操作对象
- 操作后果
- 是否影响他人
- 是否影响权限

示例：

标题：从列表移除该共享日历？

说明：这只会从你的共享日历列表中移除，不会影响共享方的日历或权限设置。

按钮：取消 / 从列表移除

### 6.4 弹窗尺寸

尺寸根据任务复杂度决定。

规则：

- 简单确认框使用小弹窗。
- 权限配置使用中等弹窗。
- 详情页可以使用更大浮层或独立详情区域。
- 所有弹窗高度不得超过屏幕高度的 70%。
- 弹窗容器设置 `max-height: 70vh`，超出内容在弹窗内部滚动。
- 日程详情类浮层居中展示，默认尺寸为 `70vw × 70vh`。
- 弹窗内不要再嵌套卡片式大容器。

## 7. 列表与结果项

### 7.1 列表优先

企业检索和管理场景优先使用列表，而不是大卡片。

适用：

- 日程搜索结果
- 邮件列表
- 共享对象列表
- 权限成员列表
- 日历来源列表

### 7.2 结果项基础字段

日程结果项至少展示：

- 时间
- 标题
- 组织者

存在时展示：

- 颜色分类
- 循环标识
- 地点
- 线上会议
- 参与人
- 来源账号或日历
- 正文片段
- 附件信息

字段不存在时收起，不展示空值。

不要展示：

- 地点：无
- 附件：0
- 正文：无
- 暂无会议链接
- 未分类

### 7.3 列表布局

宽屏建议使用列式布局：

- 时间独立列。
- 标题和标签在主列。
- 组织者、参与人、地点等作为信息列或次行。
- 操作按钮在右侧。
- 普通结果行高度建议 64-88px。
- 含正文或附件片段的结果行高度最多 112px。
- 1440px 宽屏首屏至少展示 8-10 条普通结果。

窄屏可改为两行或三行：

- 第一行：时间、标题、标签。
- 第二行：组织者、参与人。
- 第三行：地点、线上会议、来源、操作。

### 7.4 加载更多

大量结果不使用无限滚动。

规则：

- 默认展示有限数量。
- 使用 `继续显示 20 条`。
- 提供 `显示全部`。
- 结果很多时提示用户缩小筛选。

## 8. 标签与徽标

### 8.1 标签类型

Coremail 常用标签：

| 类型 | 用途 | 示例 |
| --- | --- | --- |
| 颜色分类 | 用户自定义分类 | 项目、客户、重要 |
| 循环标识 | 日程重复规则 | 循环、循环 · 每周三 |
| 状态标签 | 取消、忙闲等状态 | 会议已取消、仅查看闲忙 |
| 时间提示 | 临近开始或正在进行 | 5 分钟后开始、今天 18:30 |
| 权限标签 | 日历访问权限 | 仅查看详情、可编辑 |

### 8.2 颜色分类标签

颜色分类标签必须同时展示颜色点和分类名称。

推荐：

- 蓝点 + 项目
- 绿点 + 客户
- 红点 + 重要

规则：

- 不只显示颜色点。
- 没有分类时不展示。
- 分类名称允许用户修改。
- 多分类时展示第一个，其他折叠为 `+N`。
- 在日程结果中，颜色分类放在标题后，不单独占一整行。

### 8.3 循环标识

循环标识必须配合文字。

推荐：

- 循环 · 每日
- 循环 · 每周
- 循环 · 每月第一个周一

不推荐：

- 只展示循环图标。
- 把循环当作颜色分类。
- 只写 `循环` 但不说明实际规则，除非后端确实没有重复规则字段。

### 8.4 状态标签

状态标签只展示真正的业务状态。

示例：

- 会议已取消
- 仅查看闲忙
- 无权限

规则：

- 状态标签不要挤占标题。
- 不把“我的响应”作为日程搜索列表核心展示。
- 不把会议状态和颜色分类混用。

## 9. 表单控件

### 9.1 输入框

输入框用于明确的数据输入。

规则：

- 标签说明字段意义。
- 占位文案只做提示，不替代标签。
- 必填字段需要明确。
- 错误信息靠近字段展示。

### 9.2 选择器

选择器用于有限选项。

规则：

- 当前值清晰可见。
- 多选时展示数量或摘要。
- 选项过多时支持搜索。
- 选项名称优先于头像或图标。

### 9.3 人员选择

人员选择必须优先展示姓名和部门或职位。

推荐：

- 王芳（财务部）
- 李强（产品经理）
- 陈晨（市场部）

头像可以辅助，但不能作为主要识别方式。

### 9.4 权限选择

权限选择必须展示权限名称，并在必要时展示简短说明。

示例：

- 仅查看详情：可查看标题、时间、地点和正文，不能修改日程。
- 可编辑：可查看并编辑日程，不能管理共享权限。
- 仅查看闲忙：只显示忙闲占用，不显示详细内容。

## 10. 空状态

### 10.1 空状态用途

空状态用于解释为什么没有内容，并给出下一步。

适用：

- 搜索无结果
- 未选择账号
- 暂无共享对象
- 暂无提醒

### 10.2 搜索无结果

日程搜索无结果时展示：

- 没有找到与关键词相关的日程。
- 可尝试的排查建议。

建议：

1. 扩大时间范围。
2. 调整搜索范围，例如加入共享日历或群组日历。
3. 换用组织者、参与人、地点或附件名称搜索。
4. 清除筛选条件。
5. 多账号场景检查是否选择了正确账号。

### 10.3 空状态边界

空状态不要变成营销介绍。

避免：

- 大插画
- 大段功能介绍
- 与当前任务无关的引导

## 11. Toast 与轻提示

### 11.1 使用场景

Toast 用于即时反馈。

适用：

- 保存成功
- 已复制链接
- 权限已更新
- 已从列表移除
- 操作失败

不适用：

- 解释复杂权限模型
- 替代确认框
- 展示长文本

### 11.2 文案

Toast 文案应短、具体。

推荐：

- 已从列表移除
- 权限已更新
- 当前共享日历仅可查看闲忙

不推荐：

- 操作成功
- 发生错误
- 请稍后再试

## 12. Tooltip

Tooltip 用于解释图标按钮或被截断内容。

规则：

- 文案短。
- 鼠标悬停出现。
- 不承载关键流程。
- 移动端不要依赖 tooltip 完成理解。

适用：

- 图标按钮说明
- 截断的邮箱地址
- 截断的日历名称
- 权限说明补充

## 13. 分段控件与 Tabs

### 13.1 分段控件

分段控件用于同一内容的视图切换。

适用：

- 月 / 周 / 日
- 叠加视图 / 拆分视图

规则：

- 选项数量少。
- 互斥选择。
- 当前选择明显。
- 不用于筛选多条件。

### 13.2 Tabs

Tabs 用于同一任务中的并列内容。

适用：

- 权限面板中的设置与共享记录。
- 详情中的基础信息与附件信息。

规则：

- Tab 数量不宜过多。
- Tab 切换不应丢失未保存输入。
- 不把主导航做成 Tabs。

## 14. 图标

### 14.1 图标用途

图标用于加速识别，不替代文字。

保留：

- 搜索
- 清除
- 日历
- 人员
- 地点
- 线上会议
- 附件
- 排序
- 循环
- 账号
- 更多

避免：

- AI 图标
- 纪要图标
- 保存搜索图标
- 大量头像
- 文件类型大图标
- 无业务含义的装饰图标

### 14.2 图标与文字关系

关键业务信息必须有文字。

示例：

- 循环图标 + `循环 · 每周`
- 地点图标 + `会议室 301`
- 附件图标 + `Q4预算分配表.xlsx`

不要只展示图标让用户猜。

## 15. 响应式规则

### 15.1 宽屏

宽屏优先展示高密度布局。

规则：

- 筛选项横向排列。
- 列表使用多列。
- 操作按钮靠右。
- 详情可用大浮层或右侧区域。

### 15.2 窄屏

窄屏优先保证可读和可操作。

规则：

- 筛选项允许换行或横向滚动。
- 列表行可改为多行。
- 低频筛选进入更多筛选。
- 操作按钮不要挤压标题。

### 15.3 不允许的响应式行为

避免：

- 文本重叠。
- 按钮文案被截断到不可理解。
- 搜索框位置跳层。
- 筛选项把主内容挤出首屏。
- 关键操作在窄屏消失。

## 16. 组件检查清单

新增或修改组件前，检查：

1. 这个组件解决的是哪个用户任务？
2. 它的文案是否表达真实语义？
3. 它是否和同类组件行为一致？
4. 它是否保持足够信息密度？
5. 它是否在宽屏和窄屏都可用？
6. 它是否误把筛选、状态或说明做成导航？
7. 它是否混淆颜色分类、会议状态、权限状态？
8. 它是否隐藏了组织者、参与人、来源账号等关键信息？
9. 它的危险操作是否说明影响范围？
10. 它是否能被 `docs/design.md` 和 `docs/information-architecture.md` 支撑？
