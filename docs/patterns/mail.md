# 邮箱设计模式

本文档定义 Coremail 邮箱模块的设计模式。具体邮件正文页实现规则见 `docs/features/mail-body-page.md`，邮件搜索规则见 `docs/features/mail-search.md`。

## 1. 模块定位

邮箱模块是企业用户处理沟通、资料、任务线索和协作信息的核心工作区。

邮箱不是新闻流，也不是头像驱动的社交消息页。它必须支持高密度扫读、快速处理、连续阅读、附件判断和安全风险识别。

## 2. 核心对象

邮箱模块核心对象：

- 邮件。
- 会话。
- 发件人。
- 收件人。
- 抄送人。
- 密送人。
- 附件。
- 文件夹。
- 标签。
- 旗标 / 跟进状态。
- 已读状态。
- 重要性。
- 安全状态。
- 外部发件人状态。

不要把发件人、收件人、抄送人合并成一个“人员”字段。不同关系会影响回复、回复全部、风险提示和协作判断。

## 3. 邮件列表规则

### 3.1 必显字段

邮件列表每条结果必须展示：

- 发件人。
- 主题。
- 摘要。
- 时间。

存在时展示：

- 附件。
- 旗标 / 跟进状态。
- 未读状态。
- 外部发件人。
- 安全风险。
- 标签 / 文件夹。
- 重要性。
- 关联日程。

字段不存在时收起，不展示空值。

### 3.2 布局

邮件列表优先使用高密度列表。

推荐结构：

```text
未读点 / 选择框
发件人                    状态图标 / hover 操作
主题                      时间
摘要
```

规则：

- 发件人和主题是主要扫读对象。
- 时间必须可见并靠右对齐。
- 摘要应尽量撑满一行后再截断。
- 纯列表模式下，发件人列默认只展示人名；邮箱地址进入详情、悬停补充或跨账号搜索来源说明。
- 纯列表模式下，主题与摘要列应获得最大可用宽度，并支持用户手动拖拽调整发件人、主题与摘要、状态、时间列宽。
- 关联日程如果已在主题前展示，则状态列不再重复展示关联日程图标。
- 纯列表选中邮件后，顶部操作区必须展示当前邮件的高频处理动作，并通过 `已选中 1 封`、邮件主题或同等文案说明动作对象；这些动作不能混入搜索框、文件夹标题或刷新 / 筛选等全局列表工具。右键菜单和 hover 操作不能是唯一入口。
- 多选后进入批量处理上下文，必须展示 `已选中 N 封邮件` 和批量动作。批量动作顺序建议为：标为已读、标为未读、标记旗标、归档或移动、导出、删除；危险操作放在末尾并使用危险色。`AB` 纯列表模式的批量动作放在 B 栏选择条；`ABC` 模式的 C 栏必须切换为批量处理面板。批量面板可以使用邮件堆叠动效帮助用户感知选中对象，但每封堆叠邮件必须展示完整邮件结构：头部、发件人邮箱、时间、收件关系、附件、安全提示、正文、历史引用和末尾标识；禁止只用缩略卡、摘要卡或骨架线替代邮件内容。
- 旗标、附件、安全等标识应靠近发件人或时间区域，但不能挤压主题和摘要。
- Hover 操作可以临时替换状态图标，但不能造成布局跳动。
- Hover 操作只在鼠标悬浮当前邮件行时出现，鼠标移出后消失；选中态或按钮焦点不能让 hover 操作常驻。
- 列表 hover 和右键菜单的高频动作顺序为：标为已读 / 标为未读、标记旗标 / 取消旗标、删除。
- 已读 / 未读和旗标动作的图标必须随当前状态变化；未标记旗标使用黑色线性旗标，已标记旗标使用红色面性旗标。
- 选中态使用主色弱背景和细描边，不用高饱和大面积色块。

### 3.3 禁止

禁止：

- 用大卡片展示普通邮件列表。
- 只展示头像不展示发件人。
- 隐藏时间。
- 把附件、安全、外部发件人只用无说明图标表达；纯列表状态图标必须提供可访问文字和悬停说明。
- 在列表中展示不存在字段的空值。
- 用标签堆叠挤压主题和摘要。
- 把邮箱账号名称重复展示在每封邮件内，除非跨账号搜索需要说明来源。

### 3.4 A 栏收藏夹

收藏夹是邮箱 A 栏的范围快捷入口，不是邮件对象本身。

规则：

- 收藏夹默认可展示常用账号收件箱和全账号系统文件夹，例如已发送、草稿、已删除。
- 账号收件箱收藏项打开对应账号范围；全账号收藏项打开所有账号对应文件夹，数量与列表范围必须一致。
- 收藏夹允许同一 `folderId` 来自多个账号同时存在；当同名文件夹跨账号收藏时，收藏项文本必须追加账号名称或邮箱用于区分。
- 收藏夹标题可折叠 / 展开，折叠不得改变收藏夹内容。
- 邮箱 A 栏右键菜单只展示操作项，不展示账号名、文件夹名或说明性头部。
- 邮箱 A 栏右键菜单必须按操作语义分组：状态处理、组织管理、危险操作分别成组；无对应操作的分组不展示。
- 邮箱 A 栏右键菜单不提供 `打开`。进入账号、收藏项或文件夹使用左键点击，避免右键菜单同时承担导航和管理两类语义。
- 邮箱 A 栏右键菜单、悬浮菜单和批量操作浮层必须高于布局分割线、拖拽把手和列表内容，不得被分割线穿透或遮挡。
- 账号行右键菜单提供 `创建子文件夹`、`导入归档`。
- 收藏夹项右键菜单至少提供 `从收藏夹移除`。
- 普通文件夹右键菜单至少提供 `全部标记为已读`、`创建子文件夹`、`清空文件夹`，并按收藏状态提供 `添加到收藏夹` 或 `从收藏夹移除`。
- `创建子文件夹` 和 `添加到收藏夹` 不得复用同一个图标；前者使用文件夹管理语义图标，后者使用收藏语义图标。
- `未读邮件` 和 `旗标邮件` 是派生范围，不是实体文件夹；其右键菜单只提供 `全部标记为已读`，不提供创建子文件夹、添加收藏或清空文件夹。
- `发件箱` 作为发送队列入口，不提供 A 栏右键菜单；用户应通过左键进入查看发送状态。
- `清空文件夹` 是高影响操作，必须确认操作对象、影响范围和后果后再执行。
- `从收藏夹移除` 只移除快捷入口，不删除邮件、不删除文件夹、不影响权限；文案不得写成 `删除`。
- 添加或移除收藏夹后必须有短反馈，例如 `已添加到收藏夹`、`已从收藏夹移除`。
- 收藏夹属于用户本地偏好，应在刷新和模块切换后保持。
- 收藏夹为空时提供可操作空状态，提示用户可右键文件夹添加。
- 账号下文件夹支持按住左键拖动排序，排序只影响当前账号的 A 栏展示顺序，属于用户本地偏好，应在刷新后保持。拖动过程中必须有目标行反馈，不能误触发打开文件夹。

## 4. 时间规则

有日期分组时，单封邮件行内只展示具体时间，例如 `09:20`。

无日期分组时，行内时间需要承担日期判断职责：

- 今天：`HH:mm`
- 昨天：`昨天 HH:mm`
- 本年：`M月D日`
- 跨年：`YYYY年M月D日`

详情或悬停中可展示更完整时间。

不要在企业邮箱主列表大量使用 `3分钟前`、`1小时前` 等相对时间。

## 5. 邮件正文 / 阅读窗格

邮件正文页是企业邮件处理台，不是单纯阅读页。

必须清晰展示：

- 主题。
- 发件人姓名。
- 发件人邮箱。
- 发送时间。
- 收件关系。
- 附件状态。
- 安全风险。
- 正文。
- 历史引用。
- 回复、回复全部、转发、归档、删除等操作。

阅读窗格结构建议：

1. 顶部操作区。
2. 邮件头部信息区。
3. 附件区。
4. 安全提示区。
5. 正文阅读区。
6. 历史引用区。
7. 快捷回复区。
8. 状态视图。

规则：

- 关键 metadata 必须默认可见。
- 收件人明细可折叠，但必须可展开、可键盘访问。
- 附件在存在时放在正文上方。
- 最新正文优先展示。
- 长历史引用默认折叠。
- 系统通知邮件可隐藏快捷回复，并说明不支持直接回复。

## 6. 会话邮件

会话邮件用于把同一主题或同一线程内的往来邮件组织在一起，但不能牺牲当前邮件判断。

规则：

- 列表可展示会话数量，但当前行仍必须展示最新邮件的发件人、主题、摘要和时间。
- 阅读窗格默认优先展示最新正文，历史往来折叠在下方。
- 展开历史邮件时，每封历史邮件必须保留发件人、发送时间、收件关系和附件状态。
- 回复、回复全部、转发必须针对用户当前选中的邮件或当前会话上下文，并明确收件影响范围。
- 删除单封邮件和删除整个会话不是同一种操作，必须在文案和确认中区分。

## 7. 附件规则

附件必须展示：

- 文件名。
- 文件类型。
- 文件大小。
- 安全状态。
- 预览。
- 下载。

附件状态：

- 扫描中。
- 安全。
- 有风险。
- 已阻断。
- 不可用。

规则：

- 无附件时不展示附件区。
- 附件状态必须有文字说明，不只用图标。
- 扫描中时下载禁用。
- 中风险附件下载前二次确认。
- 高风险附件可阻断下载。
- 多附件可展示前 3 个并提供查看全部。

## 8. 安全规则

安全提示分级：

- 低风险：轻提示，不阻断。
- 中风险：说明原因，关键操作二次确认。
- 高风险：阻断下载、跳转或其他高风险操作。

必须显性表达的风险：

- 外部发件人。
- 外部收件人。
- 外部链接。
- 风险附件。
- 伪造或未认证发件人。
- 敏感邮件。
- 被系统阻断的内容。
- 外部图片拦截。

不要只用颜色或图标表达安全风险。

## 9. 回复与转发

规则：

- 回复默认回复发件人。
- 回复全部涉及多人或外部人员时必须提示影响范围。
- 转发时保留原始邮件信息。
- 系统邮件可以不支持回复。
- 发送中禁止重复提交。
- 回复、转发完成后默认停留当前邮件。

回复全部保护文案应说明：

- 本次会发送给谁。
- 是否包含外部人员。
- 是否涉及敏感邮件或风险邮件。

## 10. 删除、归档与移动

语义必须准确：

- `归档`：从当前收件箱视图移走，保留邮件。
- `删除`：移入已删除或垃圾箱。
- `彻底删除`：不可恢复或恢复成本高，必须二次确认。
- `移动`：移动到指定文件夹。
- `从收件箱移除` 不等于删除。

如果当前实现暂未提供文件夹选择器，移动入口必须写明目标文件夹，例如 `移动到归档`，不能只写 `移动` 后执行归档。

删除或归档后应提供反馈；可撤销时必须提供撤销入口。

## 11. 已读、未读、跟进与星标

规则：

- 快速切换邮件不应立即标记已读。
- 停留超过合理时间或滚动正文后可标记已读。
- 手动标记已读 / 未读优先。
- 跟进、星标、旗标都属于用户主动标记，不等于系统重要性或安全风险。
- 如果产品语义使用 `旗标`，图标应使用旗子；如果产品语义使用 `星标`，图标应使用星形。不要混用同一个图标表达两种语义。
- 列表中已设置的跟进或星标需要稳定展示；hover 操作可以提供切换，但不能导致列表内容跳动。
- 如果产品语义使用 `旗标`，反馈文案必须使用 `已标记旗标`、`已取消旗标`，不能写成跟进或星标。
- 取消跟进、取消星标和标记已读都必须提供即时反馈。
- 删除或归档后默认进入下一封。
- 回复、转发、标记、移动后默认停留当前邮件。

## 12. 系统邮件

系统邮件包括系统通知、服务消息、权限通知、安全告警和自动化提醒。

规则：

- 系统邮件必须有明确标识，不能伪装成普通个人邮件。
- 系统邮件可禁用或隐藏回复、回复全部和快捷回复，但必须说明不支持直接回复。
- 系统邮件仍要展示主题、时间、来源、正文和安全状态。
- 与权限、安全、共享相关的系统邮件应提供可执行下一步，例如查看权限、查看安全说明或打开相关对象。

## 13. 状态与边界

邮件相关页面必须覆盖：

- 未选中邮件。
- 加载中。
- 正文加载失败。
- 附件加载失败。
- 邮件已删除。
- 安全拦截。
- 外部图片拦截。
- 空正文。
- 附件不可用。
- 系统邮件不可回复。
- 操作失败。

邮箱正文不设计日历式“无权限查看此邮件内容”状态。共享或公共邮箱的访问控制应体现在账号、文件夹、委托或共享入口；用户已经能在列表中看到的单封邮件，正文应进入正常、加载、错误、已删除或安全拦截等邮件语义状态。

## 14. 检查清单

实现或评审邮箱页面时，检查：

1. 列表是否展示发件人、主题、摘要、时间？
2. 摘要是否尽量撑满一行后再截断？
3. 附件、安全、外部发件人是否有文字说明？
4. 归档、删除、彻底删除、移动是否语义清楚？
5. 回复全部是否覆盖多人和外部人员风险？
6. 阅读窗格是否默认展示关键 metadata？
7. 是否覆盖加载、空、错误、已删除、安全拦截和安全风险状态？
8. 会话邮件、系统邮件、跟进/星标/旗标是否有清晰语义？
