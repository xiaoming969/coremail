# 日历设计模式

本文档定义 Coremail 日历模块的设计模式。具体日程搜索规则见 `docs/features/calendar-search.md`，提醒规则见 `docs/features/calendar-reminders.md`，共享权限规则见 `docs/features/shared-permission.md`。

## 1. 模块定位

日历模块用于浏览、创建、搜索和处理日程、会议、提醒、会议邀请、共享日历和权限。

日历页面不是单纯展示时间轴，而是企业用户判断时间、来源、组织者、参与人、地点、权限和下一步操作的工作区。

## 2. 核心对象

日历模块核心对象：

- 日历。
- 日程。
- 会议邀请。
- 组织者。
- 参与人。
- 会议室。
- 线上会议链接。
- 提醒。
- 颜色分类。
- 忙闲状态。
- 循环规则。
- 我的日历。
- 共享日历。
- 权限。

## 3. 日历主视图

日历主视图用于浏览和操作日程。

必须支持：

- 月视图。
- 周视图。
- 日视图。
- 列表视图。
- 今天。
- 上一段 / 下一段。
- 同步日历。
- 提醒。
- 新建日程。

左侧负责日历范围，顶部负责任务工具，主体负责日程查看和处理。

周视图和日视图打开、切换日期或切换叠加 / 拆分视图时，时间画布默认定位到上班开始时间，当前原型为 `08:00`。凌晨到上班前的时间仍保留在同一时间轴中，但不应让用户每次进入都先看到 00:00 到 08:00 的空白时段。

当前日期所在的周视图和日视图必须展示当前时间标尺。标尺由左侧时间轴红色时间标签和主体红色水平线组成，两者对齐；切换到非当前日期时不展示当前时间标签，避免误读为所选日期的时间。

日历画布必须引用中国节假日和调休工作日规则。休息日包含周六、周日和法定节假日；调休工作日不按周末置灰。休息日的日 / 周时间格整天使用低权重灰底，表达该日默认非工作时间；节假日全天事件仍作为节假日日历内容展示，不替代背景状态。补班日可用 `班` 文本徽标说明，但不能只靠颜色表达。

### 3.1 拆分视图表头

日 / 周拆分视图用于并列比较多个日历来源。分栏表头必须优先展示中文来源名，例如 `小华`、`张三`、`华为日历`，并用短标签说明来源类型，例如 `我的日历`、`共享日历`、`节假日日历`。邮箱地址只能作为截断内容的补充提示或详情信息，不应作为分栏主标题直接展示。

拆分视图的日期信息必须与来源名同栏稳定展示，不因长邮箱或长来源名换行挤压主体画布。拆分账号筛选 chip 同样使用中文来源名。全天区域没有日程时使用低权重空态，例如 `暂无全天日程`，不得让空态文本比真实日程更抢占注意力。

## 4. 日程搜索

日程搜索是日历模块内的普通检索状态，不是智能会议助手，也不是日历时间轴替代品。

用户搜索日程通常要：

- 找到已有印象的会议。
- 判断是否是目标会议。
- 判断会议和自己有什么关系。
- 确认时间、地点、组织者和参与人。
- 查找正文或附件线索。
- 查看详情。
- 在可入会时一键入会。

### 4.1 页面结构

推荐结构：

1. 顶部工具栏中的搜索框。
2. 筛选栏。
3. 结果数量与时间分布摘要。
4. 相关日程列表。
5. 加载更多。

主体是单一结果流。默认不把页面拆成多个大时间分组，也不模拟日历时间轴。

### 4.2 搜索框

规则：

- 搜索框位于日历顶部工具栏。
- 搜索前后位置保持一致。
- 搜索后不在主体区域重新出现大搜索框。
- 普通搜索不展示字段配置。
- 高级搜索才展示标题、正文、附件、组织者、参与人、地点等字段配置。

### 4.3 筛选与排序

筛选栏紧贴结果区域。常见筛选包括：

- 搜索范围。
- 账号。
- 时间。
- 人员。
- 会议类型。
- 颜色分类。
- 是否有附件。
- 排序。
- 更多筛选。

筛选项按业务优先级排列；搜索范围和时间通常优先。

### 4.4 结果行字段

日程搜索结果至少展示：

- 标题。
- 时间。
- 组织者。
- 参与人摘要。
- 地点或会议方式。
- 来源账号或日历。

存在时展示：

- 附件线索。
- 正文线索。
- 循环规则。
- 颜色分类。
- 会议状态。
- 可入会状态。
- 权限受限状态。

组织者和参与人必须分开表达；颜色分类和会议状态必须分开表达。

### 4.5 密度基线

搜索结果页优先高密度列表：

- 1440px 宽屏首屏至少展示 8-10 条普通结果。
- 普通结果行高度建议 64-88px。
- 含正文或附件片段的结果行高度最多 112px。
- 筛选器高度建议 36px。
- 列表操作按钮高度建议 32px 或 36px。

不要用大卡片堆叠普通结果。

## 5. 日程详情

日程详情必须完整表达会议判断所需信息。

推荐结构：

- 顶部标题和主操作。
- 时间。
- 地点或会议方式。
- 组织者。
- 参与者。
- 会议状态。
- 响应状态。
- 所属账号或日历。
- 颜色分类。
- 循环规则。
- 提醒。
- 正文或备注。
- 附件。

规则：

- 使用文档式信息行，不使用大卡片承载基础字段。
- 基础字段优先使用同一套 icon + 字段名 + 字段值结构。
- 组织者、参与人、响应、颜色分类、循环规则必须分属不同字段。
- 桌面端可使用两列字段矩阵提高空间利用率；参与者和正文可跨列展示。
- 列表和详情页只保留用户真正需要的入会入口。
- 不展示“主持人入口”这类内部或低频入口。
- 关闭详情后回到打开它的来源上下文。
- 日程详情作为任务浮层打开，不替换底层日历画布或搜索结果。用户从日历画布查看详情时，底层仍保持原周 / 日 / 月视图、滚动位置和选中上下文；用户从提醒查看详情时，应关闭提醒窗口，让详情浮在原日历画布之上。

## 6. 创建与编辑日程

创建与编辑日程是日历模块内的任务流程。

入口：

- 新建日程按钮。
- 日历空白时间段。
- 邮件生成日程。
- 复制或编辑已有日程。

规则：

- 默认继承当前账号或日历上下文。
- 时间、标题、参与人、地点或会议方式是核心字段。
- 单个日程提醒时间优先于全局默认提醒设置。
- 编辑周期日程时，应让用户明确修改单次还是整个系列。
- 保存后回到打开来源。

## 7. 会议邀请

会议邀请是日历与邮箱之间的协作对象，不是普通通知。

会议邀请必须展示：

- 会议标题。
- 时间。
- 组织者。
- 参与人摘要。
- 地点或会议方式。
- 用户响应状态。
- 来源账号或日历。

操作规则：

- 接受、暂定、拒绝必须说明是否会通知组织者。
- 涉及外部组织者或外部参与人时，应显示外部人员提示。
- 已取消会议必须清楚标识，不继续展示正常入会主操作。
- 权限不足时，不展示无权查看的标题、地点、正文、附件和参与人；必须说明可见范围。
- 邀请处理后回到打开它的来源上下文，例如邮件、提醒、日历详情或搜索结果。

## 8. 提醒

提醒规则以减少无效打扰为目标。

核心规则：

- 全局关闭提醒弹窗时，不展示任何日历提醒弹窗。
- 当前用户最终提醒时间为 `不提醒` 时，不展示提醒。
- 未来会议到达提醒时间后正常提醒。
- 正在进行中的会议仍可提醒。
- 已结束会议不展示提醒。
- 用户在当前客户端点击加入会议后，该日程实例后续不再重复提醒。
- 单个日程提醒设置优先级最高。

提醒弹窗应提供：

- 日程标题。
- 时间。
- 地点或会议链接。
- 所属日历。
- 加入会议。
- 稍后提醒。
- 清除。
- 提醒设置入口。

详细规则见 `docs/features/calendar-reminders.md`。

## 9. 颜色分类、状态与循环

颜色分类用于用户或组织对日程做颜色标记，不等于会议状态。

必须区分：

- 颜色分类。
- 会议状态。
- 忙闲状态。
- 用户响应。
- 权限状态。

循环会议必须展示文字，例如 `循环 · 每周三`。不要只展示循环图标。若已知日程日期，周循环应展示到星期几，月循环应展示到具体日期；只有后端缺少规则字段时才退化为 `循环 · 每周` 或 `循环 · 每月`。

会议取消、草稿、仅忙闲、私密等状态必须用文字说明，不只靠颜色。

## 10. 共享日历与权限

我的日历和共享日历必须区分。

我的日历可提供：

- 颜色设置。
- 共享与权限。
- 默认提醒或显示设置。
- 删除日历。

共享日历可提供：

- 颜色设置。
- 查看权限。
- 从列表移除。

共享日历不提供：

- 删除日历。
- 共享与权限。
- 修改自己被授予的权限。

必须区分：

- 删除我拥有的日历。
- 从我的列表移除别人共享给我的日历。
- 取消我共享给别人的权限。
- 修改别人对我日历的访问权限。

详细规则见 `docs/patterns/permission.md` 和 `docs/features/shared-permission.md`。

## 11. 权限受限日程

权限受限时，界面应明确说明可见范围。

示例：

- 仅查看忙闲：隐藏标题、地点、正文、附件和参与人。
- 仅查看详情：可查看详细事件信息，不可编辑。
- 可编辑：可查看并编辑日程，但不可管理共享权限。

不要把无权限字段显示为空值。应明确说明权限不足或直接收起敏感字段。

## 12. 检查清单

实现或评审日历页面时，检查：

1. 左侧范围、顶部任务、主体判断是否清楚？
2. 搜索是否仍属于日历模块内状态？
3. 搜索结果是否以高密度列表为主？
4. 日程详情是否使用信息行而不是大卡片？
5. 组织者、参与人、来源、状态、颜色分类是否分开表达？
6. 提醒是否区分未来、进行中、已结束和用户点击加入会议？
7. 共享日历是否没有删除和共享管理入口？
8. 权限不足时是否有明确说明？
9. 会议邀请是否展示响应状态、通知后果和外部人员风险？
