# 权限、共享与可见性设计模式

本文档定义 Coremail 中权限、共享、委托和可见性的通用设计规则。共享日历的具体规格见 `docs/features/shared-permission.md`。

## 1. 适用范围

权限规则横跨：

- 共享日历权限。
- 共享邮箱 / 委托邮箱。
- 联系人可见范围。
- 邮件正文和附件访问。
- 日程可见范围。
- 只读 / 可编辑 / 仅查看忙闲。
- 删除、移除、取消共享、撤销访问。

## 2. 核心原则

1. 所有危险或有后果操作必须说明影响范围。
2. 不能把“移除入口”和“删除对象”混用。
3. 不能把“取消共享”和“删除数据”混用。
4. 权限不足必须有明确状态说明和可行下一步。
5. 不展示用户无权查看的敏感字段。
6. 数据判断使用稳定权限 ID，不使用中文展示文案。

## 3. 权限类型

通用权限语义：

| 权限 | 含义 |
| --- | --- |
| 仅查看忙闲 | 只能看到占用时间，不可查看标题、地点、正文、附件和参与人 |
| 仅查看详情 | 可查看详细信息，不可编辑 |
| 可编辑 | 可查看并编辑对象内容，不可自动获得共享管理权 |
| 管理权限 | 可管理共享对象、成员或权限 |
| 无权限 | 不可查看或不可执行当前操作 |

规则：

- 可编辑不等于可管理共享权限。
- 仅查看详情不等于可转发、下载或再次共享。
- 权限标签必须有简短说明。

## 4. 删除、移除、取消共享

必须区分：

| 操作 | 语义 |
| --- | --- |
| 删除对象 | 删除我拥有的邮件、日历、联系人或其他数据对象 |
| 彻底删除 | 从可恢复区域移除，通常不可恢复或恢复成本高 |
| 从列表移除 | 只从我的入口或可见列表中移除，不删除共享方对象 |
| 取消共享 | 共享方撤销某个对象的访问权限 |
| 修改权限 | 改变被共享对象的可见或可编辑范围 |

禁止：

- 把共享日历的接收方操作写成 `删除日历`。
- 把取消共享写成 `删除权限`。
- 把从列表移除写成 `取消共享`。
- 把删除对象写成轻量无后果操作。

## 5. 共享日历

我的日历：

- 可以共享。
- 可以修改权限。
- 可以取消共享。
- 可以删除日历。

共享日历：

- 可以查看权限。
- 可以从列表移除。
- 不可删除共享方日历。
- 不可修改自己被授予的权限。
- 不可再次共享，除非明确授予管理权限。

从列表移除必须说明：

```text
这只会从你的共享日历列表中移除，不会影响共享方的日历或权限设置。
```

## 6. 共享邮箱与委托邮箱

共享邮箱或委托邮箱必须说明：

- 当前邮箱来源。
- 当前用户身份。
- 可执行操作。
- 是否可代表他人发送。
- 是否可查看、移动、删除或归档邮件。
- 是否影响共享邮箱中的其他成员。

涉及代表他人发送、删除共享邮箱邮件、移动共享邮箱文件夹时，必须说明影响范围。

## 7. 联系人可见范围

通讯录和人员选择需要根据权限展示：

- 内部员工。
- 外部联系人。
- 部门成员。
- 群组成员。
- 隐藏人员。
- 不可选人员。

权限不足时，不要展示空姓名或空邮箱。应展示明确状态，例如：

```text
你暂无权限查看该联系人详情。
```

并给出可行下一步，例如联系管理员或更换搜索范围。

## 8. 邮件与附件权限

邮件正文、附件、外部图片和链接可能受权限或安全策略限制。

规则：

- 权限不足时说明不能查看正文或附件。
- 附件被阻断时说明阻断原因。
- 外部图片被拦截时说明隐私保护原因。
- 高风险附件或链接可阻断操作。
- 下载、打开、转发高风险内容前可二次确认。

## 9. 权限不足状态

权限不足状态必须包含：

- 当前不可执行什么。
- 为什么不可执行。
- 用户可以做什么下一步。

示例：

```text
你暂无权限查看此邮件内容。
如果需要访问，请联系邮箱管理员或邮件所有者。
```

```text
当前共享日历仅可查看忙闲，无法查看事件详情。
```

## 10. 检查清单

实现或评审权限相关功能时，检查：

1. 操作对象是谁？
2. 操作是否影响他人？
3. 文案是否区分删除、移除、取消共享和修改权限？
4. 权限不足是否有明确原因和下一步？
5. 无权限字段是否被隐藏或用状态说明替代？
6. 可编辑是否被误写成可管理共享权限？
7. 接收方是否不能管理共享方对象？
